# TermCat Client 正式版编译配置指南

## 1. 环境变量配置

项目通过 Vite 的 `import.meta.env` 机制注入环境变量。所有自定义变量必须以 `VITE_` 前缀开头。

### 1.1 创建 `.env` 文件

在 `termcat_client/` 根目录下创建 `.env` 文件（参考 `.env.example`）：

```bash
cp .env.example .env
```

### 1.2 环境变量清单

| 变量名 | 用途 | 代码中的默认值 | 正式环境示例 |
|--------|------|---------------|-------------|
| `VITE_API_BASE_URL` | REST API 地址 | `http://localhost:8080/api/v1` | `https://api.termcat.com/api/v1` |
| `VITE_WS_BASE_URL` | WebSocket 地址 | `ws://localhost:8080` | `wss://api.termcat.com` |
| `VITE_AUTH_URL` | 认证页面地址 | `http://localhost:5174` | `https://auth.termcat.com` |

### 1.3 正式环境 `.env` 文件示例

```env
VITE_API_BASE_URL=https://api.termcat.com/api/v1
VITE_WS_BASE_URL=wss://api.termcat.com
VITE_AUTH_URL=https://auth.termcat.com
```

### 1.4 多环境配置（可选）

Vite 支持按环境加载不同 `.env` 文件：

| 文件 | 加载时机 |
|------|---------|
| `.env` | 所有环境 |
| `.env.development` | `npm run dev` 时 |
| `.env.production` | `npm run build` 时 |

例如创建 `.env.production`：

```env
VITE_API_BASE_URL=https://api.termcat.com/api/v1
VITE_WS_BASE_URL=wss://api.termcat.com
VITE_AUTH_URL=https://auth.termcat.com
```

这样开发时使用 `.env.development`（或 `.env`）中的本地地址，构建时自动使用 `.env.production` 中的正式地址。

### 1.5 环境变量使用位置

| 文件 | 变量 | 说明 |
|------|------|------|
| `src/services/api.ts:10` | `VITE_API_BASE_URL` | HTTP REST API 基地址 |
| `src/services/websocket.ts:7` | `VITE_WS_BASE_URL` | 通用 WebSocket 连接 |
| `src/services/aiWebSocketService.ts:55` | `VITE_WS_BASE_URL` | AI WebSocket 连接 |
| `src/components/LoginView.tsx:53` | `VITE_AUTH_URL` | 浏览器登录跳转地址 |

### 1.6 类型定义

环境变量类型定义在 `src/vite-env.d.ts`，如需新增变量需同步更新：

```typescript
interface ImportMetaEnv {
  readonly VITE_API_BASE_URL: string
  readonly VITE_WS_BASE_URL: string
  readonly VITE_AUTH_URL: string
}
```

## 2. 版本号配置

版本号的**唯一来源**是 `package.json` 的 `version` 字段。构建时 `vite.config.ts` 自动读取并通过 `define` 注入为全局常量 `__APP_VERSION__`，`src/version.ts` 从中解析出结构化版本信息。

发版时只需修改一处：

```json
// package.json
{
  "version": "0.1.1"
}
```

`src/version.ts` 无需手动修改，会自动同步。

## 3. electron-builder 打包配置

打包配置位于 `package.json` 的 `build` 字段。

### 3.1 应用标识

```json
{
  "build": {
    "appId": "com.termcat.app",
    "productName": "TermCat"
  }
}
```

正式发布时需要确认 `appId` 是否需要调整（如 `com.yourcompany.termcat`）。

### 3.2 图标

图标文件位于 `assets/termcat_icon.png`，各平台要求：

| 平台 | 格式 | 建议尺寸 |
|------|------|---------|
| macOS | `.icns` 或 `.png` | 512x512 或 1024x1024 |
| Windows | `.ico` 或 `.png` | 256x256 |
| Linux | `.png` | 512x512 |

### 3.3 各平台目标格式

```json
{
  "mac": {
    "target": ["dmg", "zip"],
    "category": "public.app-category.developer-tools",
    "icon": "assets/termcat_icon.png"
  },
  "win": {
    "target": ["nsis", "portable"],
    "icon": "assets/termcat_icon.png"
  },
  "linux": {
    "target": ["AppImage", "deb", "tar.xz"],
    "category": "Development",
    "icon": "assets/termcat_icon.png"
  }
}
```

### 3.4 macOS 代码签名（正式发布必须）

正式发布 macOS 版本需要配置代码签名和公证：

```bash
# 环境变量方式配置
export CSC_LINK="path/to/certificate.p12"
export CSC_KEY_PASSWORD="证书密码"
export APPLE_ID="your@apple.id"
export APPLE_APP_SPECIFIC_PASSWORD="app-specific-password"
export APPLE_TEAM_ID="TEAM_ID"
```

或在 `package.json` 的 `build.mac` 中配置：

```json
{
  "mac": {
    "identity": "Developer ID Application: Your Name (TEAM_ID)",
    "notarize": {
      "teamId": "TEAM_ID"
    }
  }
}
```

### 3.5 Windows 代码签名（可选）

```bash
export CSC_LINK="path/to/certificate.pfx"
export CSC_KEY_PASSWORD="证书密码"
```

## 4. 其他配置项

### 4.1 支付模式

`src/constants.ts` 中的支付模式配置：

```typescript
export const PAYMENT_CONFIG = {
  mode: 'real',  // 'real' = 真实支付（正式环境）
};
```

### 4.2 自定义协议

`src/main/main.ts` 中注册了 `termcat://` 自定义协议，用于第三方登录回调。正式环境无需额外配置。

### 4.3 开发模式判断

Main 进程通过 `process.env.VITE_DEV_SERVER_URL` 判断运行模式：
- 存在该变量 → 加载 Vite 开发服务器 URL + 自动打开 DevTools
- 不存在 → 加载打包后的 `dist/renderer/index.html`

正式构建时此变量不存在，自动走生产路径。

## 5. 编译命令

```bash
# 安装依赖
npm install

# 清理旧构建产物
npm run build:clean

# 构建全平台（在当前操作系统上构建）
npm run build

# 构建特定平台
npm run build:mac       # macOS (dmg + zip)
npm run build:win       # Windows (nsis + portable)
npm run build:linux     # Linux (AppImage + deb + tar.xz)
```

### 5.1 构建输出

```
termcat_client/
├── dist/              # Vite 编译输出
│   ├── main/          # Main 进程
│   ├── preload/       # Preload 脚本
│   └── renderer/      # 前端资源
└── release/           # electron-builder 最终安装包
    ├── TermCat-x.x.x.dmg          # macOS
    ├── TermCat-x.x.x-setup.exe    # Windows
    └── TermCat-x.x.x.AppImage     # Linux
```

## 6. 正式发布前检查清单

- [ ] `.env.production` 中配置了正确的服务器地址（HTTPS/WSS）
- [ ] `package.json` 中 `version` 已更新（唯一版本来源，自动注入到代码中）
- [ ] `package.json` 中 `appId` 确认无误
- [ ] 应用图标 `assets/termcat_icon.png` 已准备
- [ ] macOS 代码签名证书已配置（正式发布）
- [ ] `PAYMENT_CONFIG.mode` 设置为 `'real'`
- [ ] `.env` 文件已加入 `.gitignore`，不会提交到仓库
- [ ] 运行 `npm run build:clean` 清理旧产物后再构建
