# 桌面本地终端方案

> 日期：2026-03-19
> 状态：设计中

---

## 1. 背景与动机

TermCat 当前所有终端功能均基于 SSH 协议实现。macOS 和 Linux 用户可以流畅使用，但 Windows 用户必须预先配置 OpenSSH Server 才能连接本地终端，体验门槛高。

业界主流终端软件（VS Code Terminal、Hyper、Tabby、Windows Terminal）均采用 **node-pty**（底层使用 ConPTY / Unix PTY）实现本地终端，用户打开即用，零配置。

本方案在现有 SSH 远程终端之外，新增本地终端模式，并通过抽象终端层统一上层调用。

---

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| 零配置 | 用户打开即可使用本地终端，无需安装或配置任何服务 |
| 全平台 | macOS（zsh/bash）、Linux（bash/zsh）、Windows（PowerShell/CMD/WSL/Git Bash） |
| 架构解耦 | 本地终端与 SSH 终端代码独立文件，互不影响 |
| 统一抽象 | 上层组件和 Hook 通过抽象终端接口调用，不感知底层是本地还是远程 |
| 增量迭代 | 第一期仅实现纯终端功能，AI Agent / 文件浏览 / 系统监控后续迭代 |

---

## 3. 产品形态

### 3.1 两种入口

| 入口 | 场景 | 行为 |
|------|------|------|
| Dashboard 快速入口 | 用户想快速打开本地终端 | 点击按钮，自动检测默认 Shell，直接打开终端 Tab |
| Host 配置 | 用户想保存自定义本地终端配置 | 新建 Host 时选择 `connectionType: 'local'`，可配置 Shell 类型、启动目录、环境变量等 |

### 3.2 Shell 自动检测

系统自动检测当前平台上已安装的 Shell，列出供用户选择：

| 平台 | 检测策略 |
|------|---------|
| macOS | 读取 `/etc/shells`，过滤有效路径 |
| Linux | 读取 `/etc/shells`，过滤有效路径 |
| Windows | 按已知路径逐一探测：PowerShell（pwsh.exe / powershell.exe）、CMD（cmd.exe）、WSL（wsl.exe）、Git Bash（git-bash.exe / bash.exe） |

用户可在 Host 配置中指定首选 Shell，也可使用系统默认值。

### 3.3 Tab 混合显示

本地终端 Tab 与 SSH 终端 Tab 共用同一个 Tab 栏，通过图标区分：

- 本地终端 Tab：显示 `Monitor`（或 `Terminal`）图标
- SSH 终端 Tab：保持现有 `Server` 图标

## 4. 架构设计

### 4.1 核心理念：抽象终端层

引入 `ITerminalBackend` 抽象接口，将终端的"连接 / 输入 / 输出 / 调整大小 / 断开"等行为统一抽象。SSH 终端和本地终端各自以独立文件实现该接口，上层组件（XTermTerminal、useSessionManager 等）仅依赖抽象接口，不感知具体实现。

```
                    ┌─────────────────────────┐
                    │      上层（UI / Hook）     │
                    │  XTermTerminal           │
                    │  useSessionManager       │
                    │  TerminalTabBar          │
                    └───────────┬─────────────┘
                                │ 依赖抽象接口
                    ┌───────────▼─────────────┐
                    │   ITerminalBackend       │
                    │   (抽象终端接口)           │
                    └───┬───────────────┬─────┘
                        │               │
            ┌───────────▼───┐   ┌───────▼───────────┐
            │ SSHTerminal   │   │ LocalTerminal     │
            │ Backend       │   │ Backend           │
            │ (ssh-service) │   │ (local-pty-service)│
            └───────────────┘   └───────────────────┘
```

### 4.3 模块职责

| 模块 | 位置 | 职责 | 文件状态 |
|------|------|------|---------|
| `ITerminalBackend` | Renderer 层 | 抽象终端接口定义 | **新增** |
| `SSHTerminalBackend` | Renderer 层 | SSH 终端的接口实现，封装现有 SSH IPC 调用 | **新增** |
| `LocalTerminalBackend` | Renderer 层 | 本地终端的接口实现，封装 Local PTY IPC 调用 | **新增** |
| `TerminalBackendFactory` | Renderer 层 | 根据 Session 类型创建对应 Backend 实例 | **新增** |
| `local-pty-service.ts` | Main 进程 | node-pty 进程管理、Shell 检测、生命周期管理 | **新增** |
| `ssh-service.ts` | Main 进程 | 现有 SSH 服务 | **不改动** |
| `preload.ts` | Preload | 新增 `localTerminal` IPC 通道 | **扩展** |
| `XTermTerminal.tsx` | 组件 | 改为接收 `ITerminalBackend` 实例，不再直接调 SSH IPC | **重构** |
| `useSessionManager.ts` | Hook | Session 类型扩展，支持 local session | **扩展** |
| `types/index.ts` | 类型 | Host 类型扩展 connectionType | **扩展** |

## 5. 核心流程

### 5.1 快速打开本地终端

```
用户点击 Dashboard "本地终端" 按钮
  │
  ▼
Renderer: 创建 local Session（无 Host 配置）
  │
  ▼
useSessionManager: 生成 sessionId，设置 connectionType = 'local'
  │
  ▼
切换到 TerminalView，激活新 Tab
  │
  ▼
XTermTerminal 挂载
  │
  ▼
TerminalBackendFactory.create(session)
  → 检测 connectionType === 'local'
  → 创建 LocalTerminalBackend 实例
  │
  ▼
LocalTerminalBackend.connect()
  → IPC: localTerminal.create(options)
  │
  ▼
Main 进程: localPtyService.create(options)
  → 检测默认 Shell
  → pty.spawn(shell, args, { cols, rows, cwd, env })
  → 返回 ptyId
  │
  ▼
LocalTerminalBackend 注册数据监听
  → IPC event: localTerminal.onData(ptyId, data)
  → 转发给 XTermTerminal → terminal.write(data)
  │
  ▼
终端就绪，用户可交互
```

### 5.2 通过 Host 配置打开本地终端

```
用户在 Dashboard 创建 Host（connectionType = 'local'）
  → 配置 Shell 类型、启动目录、环境变量
  → 保存
  │
  ▼
用户点击 Host 的"连接"按钮
  │
  ▼
Renderer: handleConnect(host)
  │
  ▼
useSessionManager: 创建 Session（携带 Host 配置）
  │
  ▼
（后续流程与 5.1 相同，但使用 Host 中配置的 Shell / cwd / env）
```

### 5.3 用户输入流程

```
用户按键
  │
  ▼
xterm.js onData 事件
  │
  ▼
XTermTerminal: backend.write(data)
  │
  ▼
LocalTerminalBackend.write(data)
  → IPC: localTerminal.write(ptyId, data)
  │
  ▼
Main 进程: localPtyService.write(ptyId, data)
  → ptyProcess.write(data)
  │
  ▼
Shell 处理并回显
  → ptyProcess.onData(output)
  → webContents.send('local-pty-data', ptyId, output)
  │
  ▼
LocalTerminalBackend: onData 回调
  → XTermTerminal: terminal.write(output)
```

### 5.4 终端关闭流程

```
用户关闭 Tab / 点击断开
  │
  ▼
XTermTerminal: backend.disconnect()
  │
  ▼
LocalTerminalBackend.disconnect()
  → IPC: localTerminal.destroy(ptyId)
  │
  ▼
Main 进程: localPtyService.destroy(ptyId)
  → ptyProcess.kill()
  → 从 Map 中移除
  → webContents.send('local-pty-close', ptyId)
  │
  ▼
Session 清理
```

### 5.5 终端 Resize 流程

```
窗口大小变化 / Tab 切换激活
  │
  ▼
XTermTerminal: fitAddon.fit()
  → backend.resize(cols, rows)
  │
  ▼
LocalTerminalBackend.resize(cols, rows)
  → IPC: localTerminal.resize(ptyId, cols, rows)
  │
  ▼
Main 进程: localPtyService.resize(ptyId, cols, rows)
  → ptyProcess.resize(cols, rows)
```

## 6. 模块详细设计

### 6.1 ITerminalBackend（抽象终端接口）

统一的终端后端接口，定义所有终端操作的契约：

| 方法/属性 | 说明 |
|-----------|------|
| `connect(options)` | 建立连接（SSH 连接 / PTY 创建） |
| `disconnect()` | 断开连接（SSH 断开 / PTY 销毁） |
| `write(data)` | 写入用户输入 |
| `resize(cols, rows)` | 调整终端大小 |
| `onData(callback)` | 注册数据输出回调 |
| `onClose(callback)` | 注册连接关闭回调 |
| `dispose()` | 释放所有资源和监听器 |
| `id` | 连接标识符（connectionId / ptyId） |
| `type` | 后端类型：`'ssh'` / `'local'` |
| `isConnected` | 连接状态 |

### 6.2 SSHTerminalBackend

封装现有 SSH IPC 调用，实现 `ITerminalBackend`：

- `connect()` → 调用 `window.electron.sshConnect()` + `sshCreateShell()`
- `write()` → 调用 `window.electron.sshShellWrite()`
- `resize()` → 调用 `window.electron.sshShellResize()`
- `onData()` → 监听 `window.electron.onShellData()`
- `disconnect()` → 调用 `window.electron.sshDisconnect()`

本质上是对现有分散的 SSH IPC 调用做一层面向对象的封装，**不改动 ssh-service.ts**。

### 6.3 LocalTerminalBackend

封装新增 Local PTY IPC 调用，实现 `ITerminalBackend`：

- `connect()` → 调用 `window.electron.localTerminal.create()`
- `write()` → 调用 `window.electron.localTerminal.write()`
- `resize()` → 调用 `window.electron.localTerminal.resize()`
- `onData()` → 监听 `window.electron.localTerminal.onData()`
- `disconnect()` → 调用 `window.electron.localTerminal.destroy()`

### 6.4 TerminalBackendFactory

工厂类，根据 Session 的 connectionType 创建对应的 Backend：

- `connectionType === 'local'` → 创建 `LocalTerminalBackend`
- `connectionType === 'ssh' | 'direct' | 'jump'` → 创建 `SSHTerminalBackend`

### 6.5 local-pty-service.ts（Main 进程）

Main 进程中的本地 PTY 管理服务，与 ssh-service.ts 平行独立：

| 职责 | 说明 |
|------|------|
| Shell 检测 | 检测当前平台可用的 Shell 列表 |
| PTY 创建 | 调用 node-pty spawn 创建本地终端进程 |
| PTY 管理 | 维护 `Map<ptyId, PtyProcess>`，管理生命周期 |
| 数据转发 | PTY 输出 → IPC event → Renderer |
| 输入写入 | Renderer 输入 → IPC → PTY stdin |
| Resize | 转发 resize 到 PTY |
| 销毁清理 | kill 进程，清理 Map |

### 6.6 Preload 扩展

新增 `localTerminal` 命名空间，与现有 `ssh` 通道平行：

| IPC 通道 | 类型 | 说明 |
|----------|------|------|
| `local-pty-create` | invoke | 创建本地 PTY |
| `local-pty-write` | send | 写入数据 |
| `local-pty-resize` | invoke | 调整大小 |
| `local-pty-destroy` | invoke | 销毁 PTY |
| `local-pty-get-shells` | invoke | 获取可用 Shell 列表 |
| `local-pty-data` | event | PTY 输出数据 |
| `local-pty-close` | event | PTY 关闭事件 |

### 6.7 类型扩展

**Host 类型扩展**：

- `connectionType` 新增 `'local'` 选项
- 新增 `localConfig` 可选字段：
  - `shell`: 指定 Shell 路径（如 `/bin/zsh`、`pwsh.exe`）
  - `cwd`: 启动目录
  - `env`: 额外环境变量

**Session 类型扩展**：

- `connectionType` 新增 `'local'` 选项
- `connectionId` 对于本地终端存储 `ptyId`

## 7. XTermTerminal 重构策略

当前 `XTermTerminal.tsx` 直接调用 SSH IPC（`sshShellWrite`、`onShellData` 等）。重构后：

| 改动前 | 改动后 |
|--------|--------|
| 直接调 `window.electron.sshCreateShell()` | 调 `backend.connect()` |
| 直接调 `window.electron.sshShellWrite()` | 调 `backend.write()` |
| 监听 `window.electron.onShellData()` | 调 `backend.onData()` |
| 直接调 `window.electron.sshShellResize()` | 调 `backend.resize()` |
| 监听 `window.electron.onShellClose()` | 调 `backend.onClose()` |

XTermTerminal 接收 `ITerminalBackend` 实例作为 prop（或通过 Hook 注入），组件内部只与抽象接口交互。现有的 Tab 缓冲、RAF 批量写入、prompt 着色等优化逻辑不受影响。

## 8. Dashboard 快速入口

在 Dashboard 页面新增"本地终端"快速入口按钮：

- 位置：主机列表上方或侧边栏

## 9. 第一期范围

### 包含

- node-pty 集成和 local-pty-service 实现
- 全平台 Shell 自动检测
- ITerminalBackend 抽象层 + SSH / Local 两个实现
- XTermTerminal 重构为基于抽象接口
- Host 类型扩展支持 local connectionType
- Dashboard 快速入口
- Tab 混合显示 + 图标区分
- HostConfigModal 中本地终端配置 UI

### 不包含（后续迭代）

- AI Agent 适配本地终端（LocalPtyExecutor）
- 本地文件浏览器（Node.js fs API）
- 本地系统监控
- 本地终端的 CWD 自动追踪（需 OSC 7 支持检测）

## 10. 风险与对策

| 风险 | 对策 |
|------|------|
| node-pty 是原生模块，需要 electron-rebuild | 在 CI/CD 中配置 rebuild 脚本，本地开发文档说明 |
| Windows ConPTY 在旧版本 Win10 不可用 | 检测 Windows 版本，低于 1809 时提示用户升级或回退 winpty |
| node-pty 与 Electron 版本兼容性 | 锁定 node-pty 版本，CI 矩阵测试主要平台 |
| XTermTerminal 重构影响现有 SSH 功能 | SSHTerminalBackend 是对现有 IPC 的纯封装，行为不变；充分回归测试 |
| Shell 检测在不同发行版上路径差异 | 优先读 /etc/shells，Windows 按多个候选路径探测 |
