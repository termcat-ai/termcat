# 嵌套 SSH 跳转透明代理方案

> 日期：2026-04-06
> 状态：方案设计

---

## 1. 问题描述

用户通过 TermCat 连接 Host A 后，在终端内执行 `ssh user@hostB` 跳转到 Host B。此时：

- **终端 shell**：已在 Host B 上，用户输入的命令执行在 Host B
- **文件管理器**：`SSHFsHandler` 走原始 SSH session 的 SFTP → 操作的仍是 Host A 的文件
- **监控面板**：`SSHCmdExecutor(connectionId)` 走原始 SSH exec → 采集的仍是 Host A 的指标
- **AI Agent**：`hostId`/`hostName` 在连接时固定 → AI 以为自己在操作 Host A

```
┌─ 终端 ─────────────────────────────┐
│  user@hostA$ ssh user@hostB        │
│  user@hostB$ _                     │  ← shell 已在 Host B
└────────────────────────────────────┘

┌─ 文件管理器 ──────┐  ┌─ AI Agent ──────┐  ┌─ 监控面板 ──────┐
│ SSHFsHandler      │  │ hostName=hostA  │  │ SSHCmdExecutor  │
│ → SFTP on Host A  │  │ context=hostA   │  │ → cmds on Host A│
│ ❌ 应该是 Host B  │  │ ❌ 应该是 hostB │  │ ❌ 应该是 Host B│
└───────────────────┘  └─────────────────┘  └─────────────────┘
```

### 涉及场景

| 场景 | 说明 |
|------|------|
| 单次跳转 | Host A → Host B |
| 多级跳转 | Host A → Host B → Host C |
| 跳转后退出 | Host B 执行 `exit` 回到 Host A |
| 跳转失败 | `ssh hostB` 后 Connection refused / Permission denied |

---

## 2. 设计原则

### 延续现有抽象思路

TermCat 已有的分层抽象：

```
ITerminalBackend   → 屏蔽 SSH / Local 终端差异，上层不关心
IFsHandler         → 屏蔽 SSH SFTP / Local FS 差异，上层不关心
ICmdExecutor       → 屏蔽 SSH exec / Local exec 差异，上层不关心
IHostConnection    → 组合以上三者，统一对外
```

**本次新增的抽象目标：**

```
上层不关心当前在 Host A 还是 Host B
→ IHostConnection 内部透明切换 fsHandler / cmdExecutor 的委托目标
→ 文件管理器、监控面板、AI Agent 核心逻辑零改动
```

---

## 3. 整体架构

### 3.1 架构对比

```
❌ 方案一（各模块自行处理跳转 — 已否决）:

  TerminalView ──→ 检测跳转 ──→ 通知文件管理器切换 handler
                              ──→ 通知 AI Agent 更新 context
                              ──→ 通知监控面板切换 executor
                              （每个上层模块都要写切换逻辑）

✅ 方案二（IHostConnection 内部透明代理 — 采用）:

  SSHHostConnection 内部:
    NestedSSHDetector 检测跳转
      → ProxyFsHandler / ProxyCmdExecutor 自动切换委托目标
      → 上层完全无感知，调用接口不变
```

### 3.2 数据流全景

```
┌─────────────────────────────────────────────────────────────┐
│  SSHHostConnection                                          │
│                                                             │
│  ┌──────────────────┐                                       │
│  │ SSHTerminalBackend│──→ onData / onUserInput              │
│  └──────────────────┘         │                             │
│                               ▼                             │
│                    ┌─────────────────────┐                  │
│                    │ NestedSSHDetector   │                  │
│                    │ (状态机 + 跳转栈)    │                  │
│                    └────────┬────────────┘                  │
│                             │                               │
│               host-entered / host-exited                    │
│                             │                               │
│              ┌──────────────▼──────────────┐                │
│              │                             │                │
│  ┌───────────▼──────────┐  ┌──────────────▼─────────┐      │
│  │   ProxyFsHandler     │  │   ProxyCmdExecutor     │      │
│  │   implements IFsHandler  │   implements ICmdExecutor     │
│  │                      │  │                        │      │
│  │  未嵌套 → SSHFsHandler  │  未嵌套 → SSHCmdExecutor      │
│  │  嵌套时 → TerminalFs │  │  嵌套时 → TerminalCmd  │      │
│  └──────────────────────┘  └────────────────────────┘      │
│                                                             │
│  对外暴露: fsHandler / cmdExecutor（上层拿到的是 Proxy）      │
└─────────────────────────────────────────────────────────────┘
        │                           │
        ▼                           ▼
  文件管理器                    监控面板
  调 fsHandler.listFiles()     调 cmdExecutor.execute()
  完全不知道底层切换了          完全不知道底层切换了
```

## 4. 模块详细设计

### 4.1 NestedSSHDetector — 跳转检测引擎

**文件：** `src/core/terminal/NestedSSHDetector.ts`

#### 数据结构

```typescript
interface NestedHost {
  hostname: string;
  username?: string;
  port?: number;
  depth: number;  // 0 = 原始 Host A, 1 = Host B, 2 = Host C ...
}

interface NestedSSHDetectorEvents {
  'host-entered': (host: NestedHost, stack: NestedHost[]) => void;
  'host-exited':  (current: NestedHost | null, stack: NestedHost[]) => void;
}
```

#### 状态机

```
                      检测到 ssh 命令
        IDLE ─────────────────────────→ PENDING_LOGIN
         ▲                                   │
         │                          ┌────────┴────────┐
         │                          ▼                  ▼
         │                   检测到登录成功        检测到失败
         │                          │            (refused/denied/timeout)
         │                          ▼                  │
         │                       NESTED ◄──────────────┘ (cancel, 回 IDLE)
         │                          │
         │                   检测到 exit/logout
         │                          │
         │                   ┌──────▼──────┐
         │                   │ stack.pop() │
         │                   └──────┬──────┘
         │                          │
         │              ┌───────────┴───────────┐
         │              ▼                       ▼
         └──── stack 空 (回到原始 host)   stack 非空 (回到上一级 host)
                                                │
                                                └──→ NESTED (继续)
```

#### 检测规则

| 阶段 | 信号 | 方法 |
|------|------|------|
| SSH 发起 | 用户输入匹配 `ssh [-p \d+] [user@]host` | `feedInput()` 正则匹配 |
| 登录成功 | 终端输出包含 `Last login:`、`Welcome to`、或 prompt hostname 变化 | `feedOutput()` 模式匹配 |
| 登录失败 | 输出 `Connection refused`、`Permission denied`、`No route to host`、`Connection timed out` | `feedOutput()` 模式匹配 |
| 退出跳转 | 用户输入 `exit`、`logout`，或发送 `\x04` (Ctrl+D) | `feedInput()` 匹配 |
| 退出确认 | 输出 `Connection to ... closed`，或 prompt hostname 回退 | `feedOutput()` 确认 |

#### 关键接口

```typescript
class NestedSSHDetector {
  /** 喂入用户键盘输入（原始字节流，需要攒行） */
  feedInput(data: string): void;

  /** 喂入终端输出流 */
  feedOutput(data: string): void;

  /** 当前实际 host（null = 未嵌套，在原始 host 上） */
  get currentHost(): NestedHost | null;

  /** 是否处于嵌套状态 */
  get isNested(): boolean;

  /** 当前嵌套深度 */
  get depth(): number;

  /** 完整跳转栈 */
  get hostStack(): readonly NestedHost[];

  /** 事件订阅 */
  on(event: 'host-entered', cb: ...): void;
  on(event: 'host-exited', cb: ...): void;
  off(event: string, cb: ...): void;

  dispose(): void;
}
```

### 4.2 TerminalCmdExecutor — 终端命令执行器

**文件：** `src/core/terminal/TerminalCmdExecutor.ts`

嵌套 SSH 后，无法通过原始 SSH session 的 exec 通道在 Host B 上执行命令。需要通过终端 stdin/stdout 执行命令并捕获输出。

#### 核心协议

```
发送到终端 stdin:
  echo '===TERMCAT_CMD_START_{uuid}==='; {command} 2>&1; echo "===TERMCAT_CMD_EXIT_$?_{uuid}==="; echo '===TERMCAT_CMD_END_{uuid}==='

监听终端 stdout:
  截取 START marker 和 END marker 之间的内容
  从 EXIT marker 解析 exit code
```

#### 接口

```typescript
class TerminalCmdExecutor implements ICmdExecutor {
  constructor(private terminal: ITerminalBackend) {}

  async execute(command: string): Promise<CmdResult> {
    // 1. 生成唯一 uuid
    // 2. 注册输出监听器（临时 hook terminal.onData）
    // 3. 发送 marker 包裹的命令到 terminal.write()
    // 4. 等待 END marker 出现（带超时）
    // 5. 解析输出和 exit code
    // 6. 清理监听器
    // 7. 返回 { output, exitCode }
  }
}
```

#### 注意事项

- 需要排队机制：同时只能有一个命令在执行（串行队列）
- 超时保护：命令执行超时后返回 timeout 错误
- Marker 需要足够独特，避免与正常输出混淆
- 执行期间终端的正常输出会被 marker 包裹的内容污染，需要抑制回显或事后清理

### 4.3 TerminalFsHandler — 终端文件操作

**文件：** `src/core/terminal/TerminalFsHandler.ts`

基于 `TerminalCmdExecutor` 实现 `IFsHandler` 接口，通过 shell 命令模拟 SFTP 操作。

#### 方法实现映射

| IFsHandler 方法 | 终端命令 | 说明 |
|-----------------|---------|------|
| `listFiles(path)` | `ls -la --time-style=long-iso {path}` | 解析 ls 输出为 FileItem[] |
| `getDirectoryTree(path, depth)` | `find {path} -maxdepth {depth} -type d` | 构建树结构 |
| `getFileContent(path, maxLines)` | `head -n {maxLines} {path}` | 预览 |
| `readFileForEdit(path, maxSizeKB)` | `cat {path} \| base64` | base64 编码传输，避免特殊字符 |
| `writeFileContent(path, content)` | `echo '{base64}' \| base64 -d > {path}` | base64 解码写入 |
| `writeFileContentSudo(path, content, pwd)` | `echo '{pwd}' \| sudo -S tee {path}` | sudo 写入 |
| `rename(dir, old, new)` | `mv {dir}/{old} {dir}/{new}` | |
| `deleteFile(dir, name, isDir)` | `rm -rf {dir}/{name}` 或 `rm {dir}/{name}` | |
| `mkdir(dir, name)` | `mkdir -p {dir}/{name}` | |
| `createFile(dir, name)` | `touch {dir}/{name}` | |
| `chmod(dir, name, octal)` | `chmod {octal} {dir}/{name}` | |
| `getInitialPath()` | `echo $HOME` | |
| `getTerminalCwd()` | `pwd` | |
| `packFiles(dir, names)` | `tar czf /tmp/termcat_pack_{uuid}.tar.gz -C {dir} {names}` | |
| `downloadFile` | 不支持（嵌套 SSH 无法直传文件到本地） | 返回提示 |
| `uploadFile` | 不支持（同上） | 返回提示 |

#### 不支持的操作

嵌套 SSH 场景下，以下操作无法直接实现：

- **downloadFile / downloadDirectory**：SFTP 不可达 Host B，无法直传文件到本地
- **uploadFile / uploadDirectory**：同上

这些操作在 Proxy 层返回明确的错误提示，告知用户嵌套 SSH 下不支持文件传输，建议使用 `scp` 或配置直连。

### 4.4 ProxyCmdExecutor — 命令执行代理

**文件：** `src/core/terminal/ProxyCmdExecutor.ts`

```typescript
class ProxyCmdExecutor implements ICmdExecutor {
  private _sshExecutor: SSHCmdExecutor;
  private _terminalExecutor: TerminalCmdExecutor | null = null;
  private _isNested = false;

  constructor(sshExecutor: SSHCmdExecutor) {
    this._sshExecutor = sshExecutor;
  }

  async execute(command: string): Promise<CmdResult> {
    if (this._isNested && this._terminalExecutor) {
      return this._terminalExecutor.execute(command);
    }
    return this._sshExecutor.execute(command);
  }

  /** 切换到嵌套模式（内部方法，仅 SSHHostConnection 调用） */
  _switchToNested(termExecutor: TerminalCmdExecutor): void {
    this._terminalExecutor = termExecutor;
    this._isNested = true;
  }

  /** 恢复原始模式 */
  _switchToOriginal(): void {
    this._isNested = false;
    this._terminalExecutor = null;
  }
}
```

### 4.5 ProxyFsHandler — 文件操作代理

class ProxyFsHandler implements IFsHandler {
  private _sshHandler: SSHFsHandler;

  private _isNested = false;

  async rename(dirPath: string, oldName: string, newName: string) { return this._active.rename(dirPath, oldName, newName); }
  async deleteFile(dirPath: string, name: string, isDir: boolean) { return this._active.deleteFile(dirPath, name, isDir); }

  async removeTempFile(tempPath: string) { return this._active.removeTempFile(tempPath); }
  async downloadFile(remotePath: string, localPath: string) { return this._active.downloadFile(remotePath, localPath); }
  async downloadDirectory(remotePath: string, localPath: string) { return this._active.downloadDirectory(remotePath, localPath); }

### 4.6 SSHHostConnection 集成改造

**文件：** `src/core/terminal/SSHHostConnection.ts`（修改）

```typescript
import { NestedSSHDetector } from './NestedSSHDetector';
import { TerminalCmdExecutor } from './TerminalCmdExecutor';
import { TerminalFsHandler } from './TerminalFsHandler';
import { ProxyCmdExecutor } from './ProxyCmdExecutor';
import { ProxyFsHandler } from './ProxyFsHandler';

export class SSHHostConnection implements IHostConnection {
  readonly type: HostConnectionType = 'ssh';

  private _id: string = '';
  private _terminal: SSHTerminalBackend | null = null;
  private _proxyFs: ProxyFsHandler | null = null;
  private _proxyCmd: ProxyCmdExecutor | null = null;
  private _detector: NestedSSHDetector | null = null;
  private _effectiveHostname: string | null = null;
  private _hostChangedCallbacks: Array<(hostname: string) => void> = [];
  private _isConnected = false;

  constructor(private host: Host) {}

  get id(): string { return this._id; }
  get terminal(): SSHTerminalBackend | null { return this._terminal; }

  // 上层拿到的是 Proxy，自动路由
  get fsHandler(): IFsHandler | null { return this._proxyFs; }
  get cmdExecutor(): ICmdExecutor | null { return this._proxyCmd; }

  // 新增：当前实际 host
  get effectiveHostname(): string | null {
    return this._effectiveHostname ?? this.host.hostname;
  }

  // 新增：订阅 host 变化（仅 UI 展示用）
  onHostChanged(cb: (hostname: string) => void): () => void {
    this._hostChangedCallbacks.push(cb);
    return () => {
      this._hostChangedCallbacks = this._hostChangedCallbacks.filter(c => c !== cb);
    };
  }

  async connect(): Promise<void> {
    const session = await sshService.connect(this.host);
    if (!session.connectionId) throw new Error('SSH connection failed: no connectionId');

    this._id = session.connectionId;
    this._terminal = new SSHTerminalBackend(session.connectionId, this.host.terminal?.encoding);

    // 原始 handler（Host A）
    const sshFs = new SSHFsHandler(session.connectionId);
    const sshCmd = new SSHCmdExecutor(session.connectionId);

    // 用 Proxy 包装，对外暴露的始终是 Proxy
    this._proxyFs = new ProxyFsHandler(sshFs);
    this._proxyCmd = new ProxyCmdExecutor(sshCmd);

    // 初始化跳转检测
    this._detector = new NestedSSHDetector();
    this._setupDetectorHooks();

    this._isConnected = true;
  }

  private _setupDetectorHooks(): void {
    if (!this._detector || !this._terminal) return;

    // 接入终端数据流
    this._terminal.onData(data => this._detector!.feedOutput(data));
    // 拦截用户输入（需要在 terminal.write 前 hook）
    // 具体实现取决于 SSHTerminalBackend 是否提供 onUserInput 钩子

    // 跳转进入
    this._detector.on('host-entered', (nestedHost) => {
      const termCmd = new TerminalCmdExecutor(this._terminal!);
      const termFs = new TerminalFsHandler(termCmd);
      this._proxyCmd!._switchToNested(termCmd);
      this._proxyFs!._switchToNested(termFs);
      this._effectiveHostname = nestedHost.hostname;

      // 通知 UI
      this._hostChangedCallbacks.forEach(cb => cb(nestedHost.hostname));

      log.info('nested-ssh.entered', 'Nested SSH detected', {
        hostname: nestedHost.hostname,
        depth: nestedHost.depth,
      });
    });

    // 跳转退出
    this._detector.on('host-exited', (current) => {
      if (current) {
        // 回到上一级嵌套 host
        const termCmd = new TerminalCmdExecutor(this._terminal!);
        const termFs = new TerminalFsHandler(termCmd);
        this._proxyCmd!._switchToNested(termCmd);
        this._proxyFs!._switchToNested(termFs);
        this._effectiveHostname = current.hostname;
      } else {
        // 回到原始 Host A
        this._proxyCmd!._switchToOriginal();
        this._proxyFs!._switchToOriginal();
        this._effectiveHostname = null;
      }

      const hostname = current?.hostname ?? this.host.hostname;
      this._hostChangedCallbacks.forEach(cb => cb(hostname));

      log.info('nested-ssh.exited', 'Returned from nested SSH', {
        hostname,
        depth: current?.depth ?? 0,
      });
    });
  }

  dispose(): void {
    this._detector?.dispose();
    this._terminal?.dispose();
    if (this._isConnected) this.disconnect();
    this._terminal = null;
    this._detector = null;
  }
}
```

### 4.7 IHostConnection 接口变更

**文件：** `src/core/terminal/IHostConnection.ts`（修改）

```typescript
export interface IHostConnection {
  // ── 现有接口，不变 ──
  readonly type: HostConnectionType;
  readonly id: string;
  readonly terminal: ITerminalBackend | null;
  readonly fsHandler: IFsHandler | null;      // 内部已是 Proxy，自动路由
  readonly cmdExecutor: ICmdExecutor | null;   // 内部已是 Proxy，自动路由
  dispose(): void;

  // ── 新增（仅供 UI 展示用，业务逻辑不依赖） ──

  /** 当前实际操作的 hostname（未嵌套时返回原始 hostname） */
  readonly effectiveHostname: string | null;

  /**
   * 订阅 host 变化事件（可选实现）
   * 仅用于 UI 展示（tab 标题、面板标题等），业务逻辑不依赖此事件。
   * 返回取消订阅函数。
   */
  onHostChanged?(cb: (hostname: string) => void): () => void;
}
```

**上层模块影响：**

| 模块 | 是否需要改代码 | 说明 |
|------|--------------|------|
| 文件管理器 | **否** | 调用 `fsHandler.listFiles()` 不变，Proxy 自动路由 |
| 监控面板 | **否** | 调用 `cmdExecutor.execute()` 不变，Proxy 自动路由 |
| AI Agent (associated) | **否** | 命令写到终端 stdin，已在 Host B 上执行 |
| AI Agent (hostName) | **可选** | 若需要 AI 知道在 Host B，订阅 `onHostChanged` 更新上下文 |
| TerminalView (UI) | **小改** | 订阅 `onHostChanged` 更新 tab 标题 |

## 6. 限制与 Fallback

### 6.1 文件传输不可达

嵌套 SSH 下 `TerminalFsHandler` 无法支持文件上传/下载（无 SFTP 通道到 Host B）。

处理方式：
- `downloadFile` / `uploadFile` / `downloadDirectory` / `uploadDirectory` 抛出描述性错误
- 文件管理器 UI 显示提示："嵌套 SSH 连接下不支持文件传输，请配置直连或使用 scp 命令"

### 6.2 终端命令执行的局限

`TerminalCmdExecutor` 通过 stdin/stdout marker 协议执行命令，存在以下局限：

- **串行执行**：同一时刻只能有一个命令在跑
- **输出污染**：marker 会出现在终端输出中（需要抑制回显或清理）
- **长时间命令**：需要合理超时控制
- **交互式命令**：不支持需要交互的命令（如 `vim`、`top`）

### 6.3 检测准确性

终端输出解析存在误判风险：
- 用户 `echo "ssh user@host"` 不是真正的 SSH 跳转
- 某些 SSH 配置不输出 `Last login` 等标志
- prompt 格式不固定

Fallback 策略：检测结果仅作为**最佳猜测**，可增加手动切换按钮作为补充。
