# 主机休眠唤醒导致客户端重启问题分析

> 现象：Mac 休眠再打开时，TermCat 客户端会自动重启

## 结论

**这是调试模式（`npm run dev`）才有的问题，生产环境不受影响。**

---

## 根因分析

### 关键代码

`vite.config.ts` 中 `vite-plugin-electron` 的 `onstart` 配置：

```typescript
// vite.config.ts:13-38
electron([
  {
    entry: 'src/main/main.ts',
    // ...
    onstart({ reload }) {
      reload();  // <-- 每次重编译后都会重启整个 Electron 应用
    },
  },
  {
    entry: 'src/preload/preload.ts',
    // ...
    onstart({ reload }) {
      reload();  // <-- 同上
    },
  },
]),
```

### 事件链

1. **Mac 休眠** → Vite Dev Server 的 HMR WebSocket 连接断开
2. **Mac 唤醒** → Vite 的文件监视器（chokidar/fs.watch）可能误报文件变化
   - 休眠期间文件系统时间戳可能发生变化
   - watcher 重连时可能触发 rescan
3. `vite-plugin-electron` 检测到 main 进程或 preload 的源文件"变化"，**触发重新编译**
4. 编译完成后调用 `onstart({ reload })`，`reload()` **重启整个 Electron 应用**（关闭当前窗口，重新启动进程）

### 为什么生产模式不受影响

打包后的应用（`npm run build` 产物）：
- 没有 Vite Dev Server
- 没有 HMR 热更新
- 没有文件监视器

因此不存在此问题。

---

## 可选缓解方案

### 方案 1：将 `reload()` 改为 `startup()`

```typescript
onstart(args) {
  args.startup();  // 只在进程未启动时启动，不重载已有窗口
}
```

> 注意：`startup()` vs `reload()` 的行为取决于 `vite-plugin-electron` 的版本，需要测试验证。

### 方案 2：接受现状

这是开发模式的正常行为，不影响生产环境，可以不做处理。
