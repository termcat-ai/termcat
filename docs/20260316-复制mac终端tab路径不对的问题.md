# 复制 Mac 终端 Tab 路径不对的问题

**日期**: 2026-03-16
**状态**: 已修复

## 问题描述

连接 macOS 本机（127.0.0.1），在终端中 `cd /Users/dum/Vmware_Share` 后复制 Tab，新终端的目录是 `/Users/dum`（$HOME）而不是 `/Users/dum/Vmware_Share`。

Linux 机器不受影响。

## 根因分析

### 复制 Tab 的路径传递链路

```
1. 用户点击「复制 Tab」
   ↓
2. App.tsx:handleDuplicateSession()
   → sshPwd(session.connectionId)
   ↓
3. ssh-service.ts:getCurrentDirectory()
   → 读取 connection.currentDirectory 缓存
   → 缓存为空 → 回退到 getHomeDirectory() → exec('echo $HOME') → /Users/dum
   ↓
4. 新 session 的 initialDirectory = '/Users/dum'
   ↓
5. TerminalView shell 就绪后发送 cd /Users/dum
```

### 为什么 connection.currentDirectory 为空？

`currentDirectory` 依赖 `CommandInputArea` 的提示符正则解析来调用 `sshUpdateCwd` 更新。
解析器支持以下提示符格式：

- `user@host:/path$`（标准 bash）
- `[user@host /path]$`
- `/path $`

macOS 用的 oh-my-zsh robbyrussell 主题提示符是 `➜  dirname`，不包含完整路径，也不匹配任何正则，所以 `currentDirectory` 始终为空。

Linux 标准 bash 提示符 `user@host:path$` 能正常匹配，所以不受影响。

### 为什么不能用 exec('pwd')？

`exec` 在 SSH 上开的是新 channel，新 channel 的工作目录始终是 `$HOME`，无法获取现有交互式 shell 的实际 pwd。

## 解决方案

在 `ssh-service.ts` 的 shell 数据流中解析 **OSC 终端标题序列**，自动跟踪当前工作目录。

### 原理

oh-my-zsh 的 `lib/termsupport.zsh` 在每次 `cd` 后通过 `precmd` hook 发送窗口标题：

```
ZSH_THEME_TERM_TITLE_IDLE="%n@%m: %~"
```

实际发送的 OSC 2 序列：

```
\e]2;dum@hostname: ~/Vmware_Share\a
```

这包含了完整的路径信息（`~` 相对路径），解析后即可更新 `connection.currentDirectory`。

### 代码修改（ssh-service.ts）

1. **异步预获取 home 目录**，用于 `~` 路径展开：

```typescript
let shellHomeDir = '';
if (!isExtraShell) {
  this.getHomeDirectory(baseConnectionId).then(home => {
    shellHomeDir = home;
  }).catch(() => {});
}
```

2. **在 stream.on('data') 中解析 OSC 序列**：

```typescript
if (!isExtraShell) {
  // OSC 7: file://hostname/path — 最可靠（绝对路径）
  const osc7Match = output.match(/\x1b\]7;file:\/\/[^\/]*(\/[^\x07\x1b]*?)(?:\x07|\x1b\\)/);
  if (osc7Match) {
    connection.currentDirectory = decodeURIComponent(osc7Match[1]);
  } else {
    // OSC 2: user@host: path — oh-my-zsh 默认窗口标题
    // 注意：格式 "%n@%m: %~"，冒号后有空格，正则中用 \s* 匹配
    const osc2Match = output.match(
      /\x1b\]2;[^@\x07\x1b]+@[^:\x07\x1b]+:\s*(~[^\x07\x1b]*|\/[^\x07\x1b]*)(?:\x07|\x1b\\)/
    );
    if (osc2Match) {
      let oscPath = osc2Match[1].trim();
      if (oscPath.startsWith('~') && shellHomeDir) {
        oscPath = oscPath === '~' ? shellHomeDir : shellHomeDir + oscPath.slice(1);
      }
      if (oscPath.startsWith('/')) {
        connection.currentDirectory = oscPath;
      }
    }
  }
}
```

### 兼容性

| Shell 配置 | 路径跟踪方式 | 状态 |
|---|---|---|
| oh-my-zsh（OSC 2 标题） | 本次修复的 OSC 解析 | 已支持 |
| bash `user@host:path$` | CommandInputArea 正则 | 已支持 |
| 支持 OSC 7 的 shell | 本次修复的 OSC 7 解析 | 已支持 |
| 无标题且非标准提示符 | 回退到 $HOME | 降级 |

## 踩坑记录

首次实现时 OSC 2 正则写成了 `..+:(`，缺少冒号后的 `\s*`，导致匹配不到 oh-my-zsh 的 `user@host: ~/path`（冒号后有空格）。加上 `\s*` 后修复。
