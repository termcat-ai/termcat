# 本地终端实现系统监控方案

> 日期：2026-03-19
> 前置文档：[第二期方案](./20260319-桌面本地终端AI等能力扩展实现方案.md)
> 状态：设计中

---

## 1. 背景

当前系统监控插件（`builtin-monitoring-sidebar`）通过 `sshExecute()` 在远程服务器执行 Shell 命令（`top`、`df`、`ps`、`netstat` 等）采集指标。本地终端模式下 `connectionType === 'local'`，插件收不到 `ConnectionInfo`，监控完全不可用。

本方案目标：让系统监控在本地终端模式下可用，通过 `ICmdExecutor` 抽象统一命令执行方式。

---

## 2. 设计思路

### 2.1 ICmdExecutor — 一次性命令执行抽象

`sshExecute(connId, cmd)` 和本地 `child_process.exec(cmd)` 本质上是同一件事 —— "执行一条命令，返回 `{ output, exitCode }`"。应该有一个统一的抽象接口。

```
ICmdExecutor（新增，能力层）
├── execute(command) → { output, exitCode }
│
├── SSHCmdExecutor   → window.electron.sshExecute(connectionId, cmd)
└── LocalCmdExecutor → window.electron.localExec(cmd) → Main 进程 exec
```

这个抽象挂在 `IHostConnection` 上，和 `terminal`、`fsHandler` 同层：

```
IHostConnection
├── terminal: ITerminalBackend     ← 终端 I/O（流式）
├── fsHandler: IFsHandler          ← 文件系统操作
└── cmdExecutor: ICmdExecutor      ← 命令执行（一次性）← 新增
```

**注意**：与 `ai-agent` 的 `ICommandExecutor` 不同 ——
- `ICommandExecutor`：交互式 Shell 执行器（标记注入、输出解析、超时管理、sudo 密码），给 AI Agent 用
- `ICmdExecutor`：简单的一次性命令执行（exec 模式），给监控、文件浏览等用

### 2.2 复用现有架构

核心思路：**不改动 IOSMonitor 和三个平台实现**。它们构建的 Shell 命令在本地一样可以执行。`SystemMonitorService` 改为接收 `ICmdExecutor`，不再直接调 `sshExecute`。

### 3.1 关键决策

| 决策 | 说明 |
|------|------|
| **新增 ICmdExecutor 接口** | 统一一次性命令执行抽象，挂在 IHostConnection |
| **SystemMonitorService 接收 ICmdExecutor** | 不再直接调 `sshExecute`，通过抽象接口 |
| **SSHFsHandler 也可用 ICmdExecutor** | 替代直接调 `sshExecute`（可选，后续优化） |
| **复用 IOSMonitor** | 三个平台实现完全不变 |
| **Ping 改为 0** | 本地无网络延迟 |

## 4. 核心流程

### 4.1 本地监控启动

```
用户打开本地终端
  │
  ▼
TerminalView 推送 ConnectionInfo（connectionType='local'）
  │
  ▼
监控插件 onConnectionChange(info)
  │
  ▼
检测 connectionType === 'local'
  → 从 IHostConnection 获取 cmdExecutor（LocalCmdExecutor）
  → 通过 getPlatform() 获取 OS 类型
  → 创建 SystemMonitorService(cmdExecutor, callback, osType)
  │
  ▼
SystemMonitorService.start(3000)
  │
  ▼
每 3 秒执行:
  1. IOSMonitor.buildCommand() → 构建 Shell 命令
  2. cmdExecutor.execute(command) → 执行（不感知 SSH/Local）
  3. IOSMonitor.parseMetrics(output) → 解析
  4. 计算网速 / Ping
  5. updateCallback(metrics) → UI 更新
```

### 4.2 SSH 监控（现有逻辑迁移）

```
监控插件 onConnectionChange(info)  connectionType='ssh'
  │
  ▼
从 IHostConnection 获取 cmdExecutor（SSHCmdExecutor）
  → 创建 SystemMonitorService(cmdExecutor, callback, osType)
  │
  ▼
完全相同的轮询逻辑，只是 cmdExecutor 底层走 sshExecute
```

## 7. 重点问题

| 问题 | 解决方案 |
|------|---------|
| **Ping 指标** | SSH 模式用 RTT/2，本地模式固定为 0 |
| **OS 检测** | 本地用 `getPlatform()` 映射，SSH 用 `sshGetOSInfo()` |
| **命令兼容性** | 本地与 SSH 执行相同 Shell 命令，无需适配 |
| **安全** | `localExec` 只由内部组件使用 |
| **性能** | `child_process.exec` 异步，不阻塞 Main 进程 |
| **Windows** | `localExec` 使用 `{ shell: 'powershell.exe' }` |
| **SSHFsHandler 也可用 ICmdExecutor** | 后续优化：SSHFsHandler 改用 cmdExecutor 替代直接调 sshExecute，进一步解耦 |
