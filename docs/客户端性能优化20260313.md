## 一、启动慢（>20秒）根因分析

### 启动时序图

```
0ms     Electron app.whenReady()
        ├─ Menu 构建（macOS release 模式）
        ├─ chatHistoryService.registerHandlers()
        └─ createWindow() — BrowserWindow 创建 + preload 加载

~50ms   Window 加载 index.html
        └─ <script> 加载渲染进程 bundle

~100ms  Google Fonts 外部 CSS 请求 ← [阻塞渲染 1-3秒]
        └─ @import url('https://fonts.googleapis.com/...')

~150ms  ReactDOM.createRoot().render()
        ├─ I18nProvider: 同步加载 3 个语言包（2373 行, ~100KB）
        └─ App 组件挂载

~200ms  App.tsx useEffect 触发
        ├─ activateBuiltinPlugins()（5 个插件同步激活）
        │   └─ file-browser 插件加载 15 个 CodeMirror 语言包（~500KB+）
        ├─ authService.getUser()       [localStorage 同步读取]
        ├─ apiService.getUserProfile() [网络请求 #1]
        ├─ hostService.syncFromServer()[网络请求 #2]
        ├─ hostService.getGroups()     [localStorage + JSON.parse]
        ├─ loadProxies()               [网络请求 #3]
        ├─ fetchAIModels()             [网络请求 #4]
        └─ apiService.getLatestVersion()[网络请求 #5]

~500-5000ms  等待网络响应
        ├─ 每个 API 响应触发 setState → 全局重渲染
        ├─ getUserProfile() 被调用 2 次（重复）
        ├─ fetchAIModels() 被调用 2 次（重复）
        └─ 所有请求串行执行，无并行

~5000-20000ms  用户看到可交互界面
```

### 启动瓶颈量化

| 瓶颈 | 预估耗时 | 位置 |
|------|----------|------|
| Google Fonts 外部 CSS 加载 | 1-3s（网络依赖） | `src/renderer/styles/index.css:1` |
| 5 个内置插件同步激活 | 0.5-2s | `App.tsx:122-168` |
| CodeMirror 15 个语言包加载 | 0.3-1s | `FileEditorModal.tsx:5-27` |

| 3 个语言包全量加载 | 0.2-0.5s | `src/locales/index.ts:6-17` |
| 插件系统同步文件 I/O | 0.1-0.3s | `plugin-manager.ts:876,94-122` |
| 无代码分割，单体 bundle | 0.5-1s | `vite.config.ts` |
| 重复 API 调用 | 0.5-2s | `App.tsx:205,489(getUserProfile)`, `App.tsx:222,601(fetchAIModels)` |

**合计预估: 6-20秒**（视网络状况波动很大）

## 三、优化方案

### 阶段 0：立即可做（1 天，解决启动最大瓶颈）

#### 0.1 Google Fonts 改为本地字体

**问题**: `@import url('https://fonts.googleapis.com/...')` 阻塞渲染 1-3 秒

**方案**:
```css
/* 改前 — styles/index.css:1 */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap');

/* 改后 — 下载 Inter 字体到 assets/fonts/，只保留实际使用的字重 */
@font-face {
  font-family: 'Inter';
  font-weight: 400;
  font-display: swap;
  src: url('../assets/fonts/Inter-Regular.woff2') format('woff2');
}
@font-face {
  font-family: 'Inter';
  font-weight: 500;
  font-display: swap;
  src: url('../assets/fonts/Inter-Medium.woff2') format('woff2');
}
@font-face {
  font-family: 'Inter';
  font-weight: 600;
  font-display: swap;
  src: url('../assets/fonts/Inter-SemiBold.woff2') format('woff2');
}
@font-face {
  font-family: 'Inter';
  font-weight: 700;
  font-display: swap;
  src: url('../assets/fonts/Inter-Bold.woff2') format('woff2');
}
```

**预期收益**: 启动减少 1-3 秒，离线可用

#### 0.2 启动 API 请求并行化 + 去重

**问题**: 5 个 API 串行执行，其中 2 个有重复调用

**方案**:
```typescript
// 改前 — App.tsx useEffect 中串行调用
const profile = await apiService.getUserProfile();  // 网络 #1
// ... 处理 profile
await hostService.syncFromServer();                  // 网络 #2
// ... 处理 hosts
await loadProxies();                                 // 网络 #3
await fetchAIModels();                               // 网络 #4（重复）
const version = await apiService.getLatestVersion(); // 网络 #5

// 改后 — Promise.all 并行 + 去重
const [profile, hosts, proxies, models, version] = await Promise.allSettled([
  apiService.getUserProfile(),
  hostService.syncFromServer(),
  apiService.getProxies(),
  apiService.getAIModels(),
  shouldCheckVersion ? apiService.getLatestVersion() : Promise.resolve(null),
]);
// 再分别处理结果
```

**预期收益**: 5 次串行 → 1 次并行，网络等待时间从 sum(t1..t5) 降为 max(t1..t5)，预估减少 2-5 秒

#### 0.3 删除未使用的 socket.io-client

**问题**: `socket.io-client` 在 `package.json` 中但代码中无任何 import

**方案**:
```bash
npm uninstall socket.io-client
```

**预期收益**: 减少 ~50KB bundle 体积

### 阶段 1：高 ROI 优化（2-3 天）

#### 1.1 内置插件延迟激活

**问题**: 5 个内置插件在 App mount 时同步激活，file-browser 加载 15 个 CodeMirror 语言包

**方案**:
```typescript
// 改前 — App.tsx:122
useEffect(() => {
  activateBuiltinPlugins(); // 5 个插件全部同步激活
}, []);

// 改后 — 只激活核心插件，其余延迟
useEffect(() => {
  // 核心插件立即激活（终端 AI 面板）
  activateCorePlugins(['ai-ops']);

  // 非核心插件在 idle 时激活
  requestIdleCallback(() => {
    activateDeferredPlugins([
      'monitoring-sidebar',
      'file-browser',      // CodeMirror 延迟加载
      'transfer-manager',
      'command-library',
    ]);
  });
}, []);
```

**预期收益**: 首屏渲染减少 0.5-2 秒

#### 1.2 CodeMirror 语言包动态导入

**问题**: `FileEditorModal.tsx:5-27` 静态导入 15 个语言包（~500KB+）

**方案**:
```typescript
// 改前 — FileEditorModal.tsx
import { javascript } from '@codemirror/lang-javascript';
import { json } from '@codemirror/lang-json';
import { python } from '@codemirror/lang-python';
// ... 15 个 import

// 改后 — 按需动态导入
const languageLoaders: Record<string, () => Promise<Extension>> = {
  javascript: () => import('@codemirror/lang-javascript').then(m => m.javascript()),
  json: () => import('@codemirror/lang-json').then(m => m.json()),
  python: () => import('@codemirror/lang-python').then(m => m.python()),
  // ...
};

async function getLanguageExtension(filename: string): Promise<Extension | null> {
  const ext = filename.split('.').pop()?.toLowerCase();
  const loader = languageLoaders[mapExtToLanguage(ext)];
  return loader ? await loader() : null;
}
```

**预期收益**: 初始 bundle 减少 ~500KB，文件编辑器打开时才加载对应语言

#### 1.3 I18n 按需加载

**问题**: `locales/index.ts` 静态导入 3 个语言包，实际只用 1 个

**方案**:
```typescript
// 改前 — locales/index.ts
import { zhTranslations } from './zh';
import { enTranslations } from './en';
import { esTranslations } from './es';

// 改后 — 动态加载
const loaders = {
  zh: () => import('./zh').then(m => m.zhTranslations),
  en: () => import('./en').then(m => m.enTranslations),
  es: () => import('./es').then(m => m.esTranslations),
};

export async function getTranslation(lang: string) {
  const loader = loaders[lang] || loaders['zh'];
  return await loader();
}
```

**预期收益**: 初始 bundle 减少 ~70KB（2 个未使用语言包）

#### 1.4 Vite 代码分割配置

**问题**: `vite.config.ts` 无 manualChunks，所有代码打包为单体 bundle

**方案**:
```typescript
// vite.config.ts — renderer build 增加分块策略
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-react': ['react', 'react-dom'],
        'vendor-xterm': ['xterm', '@xterm/addon-fit', '@xterm/addon-unicode11', '@xterm/addon-web-links'],
        'vendor-codemirror': [
          'codemirror', '@codemirror/view', '@codemirror/state',
          '@codemirror/commands', '@codemirror/language',
        ],
        'vendor-markdown': ['react-markdown'],
        'vendor-i18n': ['i18next', 'react-i18next'],
      },
    },
  },
},
```

同时对非首屏视图做 React.lazy：
```typescript
// App.tsx — 非首屏视图懒加载
const SettingsView = React.lazy(() => import('../components/SettingsView'));
const ExtensionsView = React.lazy(() => import('../components/ExtensionsView'));
const Dashboard = React.lazy(() => import('../components/Dashboard'));
```

**预期收益**: 首屏 bundle 减小 30-50%，非首屏视图按需加载

#### 1.5 插件系统异步化文件 I/O

**问题**: `plugin-manager.ts` 使用 `fs.readFileSync`/`fs.readdirSync`/`fs.existsSync` 阻塞主进程

**方案**:
```typescript
// 改前 — plugin-manager.ts:876
private loadConfig(): PluginConfigFile {
  if (fs.existsSync(this.configPath)) {
    return JSON.parse(fs.readFileSync(this.configPath, 'utf-8'));
  }
}

// 改后 — 异步版本
private async loadConfig(): Promise<PluginConfigFile> {
  try {
    const data = await fs.promises.readFile(this.configPath, 'utf-8');
    return JSON.parse(data);
  } catch {
    return { enabled: {}, settings: {} };
  }
}

// discoverPlugins 同理改为 fs.promises.readdir + fs.promises.readFile
```

**预期收益**: 主进程启动不被文件 I/O 阻塞，窗口更快出现

### 阶段 2：运行时流畅度优化（3-5 天）

#### 2.1 App.tsx 状态拆分（对应 P4）

**问题**: 21+ useState 在根组件，任一变更触发全局重渲染

**方案**: 按领域拆分为独立 Context：

```
AppProviders
├─ SessionProvider     → activeSessions, currentSessionId
├─ ThemeProvider       → theme, terminalTheme, fontSize, fontFamily
├─ UserProvider        → user, showLogin
├─ HostProvider        → hosts, groups, proxies, syncStatus, storageMode
├─ PaymentProvider     → showPaymentModal, paymentType, amount, tierId
└─ UIProvider          → isAiPanelVisible, showUpdateModal, activeView
```

每个 Provider 变更只触发订阅该 Context 的组件重渲染。

**预期收益**: 减少 70%+ 不必要重渲染

#### 2.2 后台 Tab 暂停（对应 P1 + P7 + P11）

**问题**: 所有 Tab 同时挂载，后台 Tab 的监控轮询、rAF 循环、SSH 数据处理持续消耗 CPU

**方案**:
```typescript
// TerminalView 接收 isActive prop
<TerminalView
  isActive={currentSessionId === session.id}
  // ...
/>

// XTermTerminal — 后台暂停 rAF
useEffect(() => {
  if (!isActive) {
    cancelAnimationFrame(rafId);
    return;
  }
  // 恢复焦点轮询（或改为事件驱动，见下文）
}, [isActive]);

// 更好的方案：彻底去掉 rAF 焦点轮询，改为事件驱动
terminalElement.addEventListener('focusin', handleFocus);
terminalElement.addEventListener('focusout', handleBlur);

// SystemMonitorService — 后台暂停
if (!isActive) monitorService.stop();
else monitorService.start();
```

**预期收益**: 5 Tab 场景 CPU 占用降低 60%+

#### 2.3 AI 流式输出 chunk 缓冲（对应 P2）

**问题**: 每个 ANSWER chunk（每秒 50-100 个）触发 setState → 全局重渲染 → Markdown 全文重解析

**方案**:
```typescript
// useAIMessageHandler.ts
const chunkBufferRef = useRef('');
const rafRef = useRef<number>();

function handleAnswerChunk(chunk: string) {
  chunkBufferRef.current += chunk;

  if (!rafRef.current) {
    rafRef.current = requestAnimationFrame(() => {
      const buffered = chunkBufferRef.current;
      chunkBufferRef.current = '';
      rafRef.current = undefined;

      // 一次性 flush 到 state
      setMessages(prev => {
        const last = prev[prev.length - 1];
        return [...prev.slice(0, -1), { ...last, content: last.content + buffered }];
      });
    });
  }
}
```

**预期收益**: setState 频率从 50-100次/秒 降至 ~60次/秒（与帧率对齐），每次更新包含更多内容

#### 2.4 关键组件 React.memo（对应 P8）

**问题**: MessageBubble、XTermTerminal、MonitoringSidebar 等缺少 memo

**方案**:
```typescript
// 对以下组件添加 React.memo：
export const MessageBubble = React.memo(function MessageBubble(props) { ... });
export const XTermTerminal = React.memo(function XTermTerminal(props) { ... });
export const MonitoringSidebar = React.memo(function MonitoringSidebar(props) { ... });
export const OperationPlan = React.memo(function OperationPlan(props) { ... });
export const StepDetail = React.memo(function StepDetail(props) { ... });
export const AIOpsMessages = React.memo(function AIOpsMessages(props) { ... });

// 父组件中 handler 用 useCallback 包裹
const handleConnect = useCallback((hostId: string) => {
  // ...
}, [/* 依赖 */]);
```

**预期收益**: 减少 50%+ 不必要重渲染

#### 2.5 localStorage 写入防抖（对应 P23）

**问题**: `App.tsx:406-459` 每次 state 变更立即同步写 localStorage

**方案**:
```typescript
// 改前 — 8 个独立 useEffect 各自同步写 localStorage
useEffect(() => {
  localStorage.setItem('termcat_theme', theme);
}, [theme]);

// 改后 — 批量防抖写入
const debouncedSaveSettings = useMemo(
  () => debounce((settings: Record<string, string>) => {
    Object.entries(settings).forEach(([k, v]) => localStorage.setItem(k, v));
  }, 500),
  []
);

useEffect(() => {
  debouncedSaveSettings({
    termcat_theme: theme,
    termcat_terminal_theme: terminalTheme,
    termcat_terminal_fontsize: String(terminalFontSize),
    termcat_terminal_fontfamily: terminalFontFamily,
  });
}, [theme, terminalTheme, terminalFontSize, terminalFontFamily]);
```

**预期收益**: 减少 80% 同步 I/O 次数

```typescript
  const renderTime = performance.now();
  console.log(`[Perf] First render: ${renderTime.toFixed(0)}ms`);

  // 数据加载完成计时
  Promise.allSettled([/* 所有启动请求 */]).then(() => {
    console.log(`[Perf] Data ready: ${performance.now().toFixed(0)}ms`);
  });
}, []);
```
