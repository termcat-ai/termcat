# AI 会话记录保存方案

## 1. 需求概述

为 AI 运维面板提供聊天记录的保存与查看功能：
- 用户与 AI 的对话内容按"会话"为单位持久化存储
- 会话 title 取第一条用户消息内容（截取前30字符）
- 支持查看历史会话列表、点击恢复会话内容到新 tab
- 存储路径：`~/.termcat/chat_records/<userId>/`

## 2. 核心概念定义

| 概念 | 说明 |
|------|------|
| **会话 (Conversation)** | 一次完整的 AI 对话上下文，包含多轮 user/assistant 消息 |
| **会话 ID (convId)** | 客户端生成的 UUID，标识一次会话 |
| **sessionId** | 终端 tab 的 SSH 连接 ID，一个 tab 可产生多个会话 |
| **hostId** | 目标主机 ID，用于关联会话与主机 |

## 3. 存储设计

### 3.1 目录结构

```
~/.termcat/chat_records/
  └── <userId>/
        ├── conv_<userId>_<convId>_<timestamp>.dat
        ├── conv_<userId>_<convId>_<timestamp>.dat
        └── ...
```

- `userId`：用户唯一标识，每位用户独立目录
- `convId`：会话 UUID（`crypto.randomUUID()`）
- `timestamp`：会话创建时间的 Unix 秒级时间戳

示例：
```
~/.termcat/chat_records/
  └── 123456/
        ├── conv_123456_a1b2c3d4-e5f6-7890-abcd-ef1234567890_1717577900.dat
        └── conv_123456_f9e8d7c6-b5a4-3210-fedc-ba9876543210_1717578100.dat
```

### 3.2 .dat 文件格式 — JSONL（JSON Lines）追加写入

采用 **JSONL（JSON Lines）** 格式，每行是一个独立的 JSON 对象，以 `\n` 分隔。新消息直接 `append` 到文件末尾，无需读取和重写整个文件。

文件结构分为两种行类型：**头部行（header）** 和 **消息行（message）**。

```
第1行：header 行（会话元信息）
第2行：message 行（第1条消息）
第3行：message 行（第2条消息）
...（持续追加）
```

#### 3.2.1 Header 行

文件第一行固定为 header，记录会话元信息：

```json
{"type":"header","version":1,"convId":"a1b2c3d4-e5f6-...","userId":"123456","hostId":"host-001","hostName":"prod-server-01","title":"帮我检查一下磁盘空间","mode":"ask","model":"glm-4-flash","createdAt":1717577900000,"updatedAt":1717578200000}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `"header"` | 行类型标识 |
| `version` | `number` | 格式版本号，当前为 `1` |
| `convId` | `string` | 会话 UUID |
| `userId` | `string` | 用户 ID |
| `hostId` | `string` | 关联主机 ID |
| `hostName` | `string` | 主机名称（冗余，方便列表展示） |
| `title` | `string` | 会话标题（第一条用户消息截取前30字符） |
| `mode` | `"ask" \| "agent"` | 会话模式 |
| `model` | `string` | AI 模型名称 |
| `createdAt` | `number` | 会话创建时间 (ms) |
| `updatedAt` | `number` | 最后更新时间 (ms)，每次追加消息时更新 |

#### 3.2.2 Message 行

header 之后每一行都是一条消息记录：

```json
{"type":"msg","id":"msg_a1b2c3d4","role":"user","content":"帮我检查一下磁盘空间使用情况","timestamp":1717577900000}
{"type":"msg","id":"msg_e5f6g7h8","role":"assistant","content":"好的，我来帮你检查磁盘空间...","suggestion":null,"taskState":{"taskId":"task-xxx","taskType":"operation","status":"completed","plan":[...],"tokenUsage":{"inputTokens":100,"outputTokens":200,"totalTokens":300,"costGems":1}},"timestamp":1717577905000}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | `"msg"` | 行类型标识 |
| `id` | `string` | 消息 ID |
| `role` | `"user" \| "assistant"` | 消息角色 |
| `content` | `string` | 消息文本（markdown 由 JSON.stringify 自动转义换行符） |
| `suggestion` | `AICmdSuggestion \| null` | 命令建议（可选） |
| `taskState` | `AITaskState \| null` | Agent 任务状态（可选，完整保留） |
| `files` | `FileMeta[]` | 用户附件元信息（可选，不含 base64 内容） |
| `timestamp` | `number` | 消息时间 (ms) |

#### 3.2.3 完整文件示例

```
{"type":"header","version":1,"convId":"a1b2c3d4","userId":"123456","hostId":"host-001","hostName":"prod-server-01","title":"帮我检查一下磁盘空间","mode":"ask","model":"glm-4-flash","createdAt":1717577900000,"updatedAt":1717578200000}
{"type":"msg","id":"msg_001","role":"user","content":"帮我检查一下磁盘空间使用情况","timestamp":1717577900000}
{"type":"msg","id":"msg_002","role":"assistant","content":"好的，我来帮你检查磁盘空间。\n\n```bash\ndf -h\n```\n\n以上命令可以查看所有磁盘分区的使用情况。","timestamp":1717577905000}
{"type":"msg","id":"msg_003","role":"user","content":"执行一下","timestamp":1717577910000}
{"type":"msg","id":"msg_004","role":"assistant","content":"执行完成，磁盘使用率 23%。","taskState":{"taskId":"task-xxx","taskType":"operation","status":"completed","tokenUsage":{"inputTokens":100,"outputTokens":200,"totalTokens":300,"costGems":1}},"timestamp":1717577920000}
```

#### 3.2.4 格式设计要点

1. **追加写入**：新消息只需 `fs.appendFile(filePath, JSON.stringify(msg) + '\n')`，无需读取和重写整个文件，I/O 开销最小。
2. **Header 更新**：`updatedAt` 需要更新时，只需重写文件第一行（读取全文 → 替换第一行 → 写回）。或者更简单的方案：读取会话列表时，以文件最后一条消息的 `timestamp` 作为 `updatedAt`，header 的 `updatedAt` 仅在创建时写入，后续不再更新。
3. **转义安全**：`content` 中的 markdown（反引号、换行符 `\n`、制表符等）由 `JSON.stringify` 自动转义为 `\\n` 等，保证每条 JSON 记录始终占一行。`JSON.parse` 还原时自动恢复。
4. **容错性**：读取时逐行解析，单行 JSON 损坏不影响其他行。跳过解析失败的行即可。
5. **version 字段**：当前为 `1`，未来调整格式时可按版本号做迁移。
6. **附件处理**：用户附件的 `base64Content` **不持久化**（太大），只保留 `{ name, size, type }` 元信息。
7. **taskState 完整保留**：agent 模式下的执行计划、步骤输出、token 消耗等完整序列化到消息行中。

#### 3.2.5 读写操作对比（vs 旧 JSON 方案）

| 操作 | 旧方案（整体 JSON） | 新方案（JSONL 追加） |
|------|---------------------|---------------------|
| 追加一条消息 | 读取全文 → JSON.parse → push → JSON.stringify → 全量写入 | `fs.appendFile(line + '\n')` |
| 读取完整会话 | `JSON.parse(全文)` | 逐行 `JSON.parse`，拼装 messages 数组 |
| 读取会话列表（仅 meta） | 需读取全文解析 | 只读第一行 header 即可 |
| 单条损坏恢复 | 整个文件失效 | 跳过损坏行，其余消息可恢复 |

### 3.3 类型定义

```typescript
// src/types/chatHistory.ts

/** Header 行数据结构（写入 .dat 文件第一行） */
export interface ConversationHeader {
  convId: string;           // 会话 UUID
  userId: string;           // 用户 ID
  hostId: string;           // 关联主机
  hostName: string;         // 主机名称
  title: string;            // 会话标题
  mode: 'ask' | 'agent';   // 会话模式
  model: string;            // 模型名称
  createdAt: number;        // 创建时间 (ms)
  updatedAt: number;        // 更新时间 (ms)
}

/** 会话列表项（header + 文件级信息，不含 messages） */
export interface ConversationMeta extends ConversationHeader {
  fileName: string;         // 文件名（用于加载/删除）
  fileSize: number;         // 文件大小 (bytes)
}

/** 完整会话数据（header + messages，加载时组装） */
export interface ConversationData extends ConversationHeader {
  version: number;
  messageCount: number;
  messages: AIOpsMessage[];
}
```

- `ConversationHeader`：对应 .dat 文件第一行的 JSON 内容
- `ConversationMeta`：会话列表展示用（header + 文件名），不含 messages
- `ConversationData`：完整会话数据（加载时逐行解析拼装）

## 4. IPC 通道设计

文件系统操作必须在 Main 进程执行，Renderer 通过 IPC 调用。

### 4.1 Main 进程服务

```typescript
// src/main/services/chat-history-service.ts

import { app, ipcMain } from 'electron';
import * as fs from 'fs/promises';
import * as fSync from 'fs';
import * as path from 'path';
import * as readline from 'readline';

class ChatHistoryService {
  private getBasePath(): string {
    return path.join(app.getPath('home'), '.termcat', 'chat_records');
  }

  private getUserDir(userId: string): string {
    return path.join(this.getBasePath(), userId);
  }

  private getFilePath(userId: string, convId: string, createdAt: number): string {
    const ts = Math.floor(createdAt / 1000);
    const fileName = `conv_${userId}_${convId}_${ts}.dat`;
    return path.join(this.getUserDir(userId), fileName);
  }

  /** 创建会话（写入 header 行） */
  async createConversation(header: ConversationHeader): Promise<string> {
    const dir = this.getUserDir(header.userId);
    await fs.mkdir(dir, { recursive: true });
    const filePath = this.getFilePath(header.userId, header.convId, header.createdAt);
    const headerLine = JSON.stringify({ type: 'header', ...header }) + '\n';
    await fs.writeFile(filePath, headerLine, 'utf-8');
    return path.basename(filePath);
  }

  /** 追加消息（append 到文件末尾） */
  async appendMessage(userId: string, convId: string, createdAt: number, message: any): Promise<void> {
    const filePath = this.getFilePath(userId, convId, createdAt);
    const line = JSON.stringify({ type: 'msg', ...message }) + '\n';
    await fs.appendFile(filePath, line, 'utf-8');
  }

  /** 批量追加消息 */
  async appendMessages(userId: string, convId: string, createdAt: number, messages: any[]): Promise<void> {
    const filePath = this.getFilePath(userId, convId, createdAt);
    const lines = messages.map(msg => JSON.stringify({ type: 'msg', ...msg })).join('\n') + '\n';
    await fs.appendFile(filePath, lines, 'utf-8');
  }

  /** 更新 header（重写第一行，保留消息行不变） */
  async updateHeader(userId: string, convId: string, createdAt: number, updates: Partial<ConversationHeader>): Promise<void> {
    const filePath = this.getFilePath(userId, convId, createdAt);
    try {
      const raw = await fs.readFile(filePath, 'utf-8');
      const firstNewline = raw.indexOf('\n');
      if (firstNewline === -1) return;
      const oldHeader = JSON.parse(raw.substring(0, firstNewline));
      const newHeader = { ...oldHeader, ...updates };
      const rest = raw.substring(firstNewline); // 包含 \n + 后续所有行
      await fs.writeFile(filePath, JSON.stringify(newHeader) + rest, 'utf-8');
    } catch {
      // 文件不存在或损坏，忽略
    }
  }

  /** 读取会话列表（只读每个文件第一行 header） */
  async listConversations(userId: string): Promise<ConversationMeta[]> {
    const dir = this.getUserDir(userId);
    try {
      const files = await fs.readdir(dir);
      const datFiles = files.filter(f => f.startsWith('conv_') && f.endsWith('.dat'));
      const metas: ConversationMeta[] = [];

      for (const fileName of datFiles) {
        try {
          // 只读第一行，不加载全部消息
          const header = await this.readFirstLine(path.join(dir, fileName));
          if (!header || header.type !== 'header') continue;

          // 用文件大小估算消息数（可选：也可计算行数）
          const stat = await fs.stat(path.join(dir, fileName));

          metas.push({
            convId: header.convId,
            userId: header.userId,
            hostId: header.hostId,
            hostName: header.hostName,
            title: header.title,
            mode: header.mode,
            model: header.model,
            createdAt: header.createdAt,
            updatedAt: header.updatedAt,
            fileName,
            fileSize: stat.size,
          });
        } catch {
          // 跳过损坏文件
        }
      }
      return metas.sort((a, b) => b.updatedAt - a.updatedAt);
    } catch {
      return [];
    }
  }

  /** 读取完整会话（逐行解析） */
  async loadConversation(userId: string, fileName: string): Promise<ConversationData | null> {
    const filePath = path.join(this.getUserDir(userId), fileName);
    try {
      const raw = await fs.readFile(filePath, 'utf-8');
      const lines = raw.split('\n').filter(line => line.trim());
      let header: any = null;
      const messages: any[] = [];

      for (const line of lines) {
        try {
          const obj = JSON.parse(line);
          if (obj.type === 'header') {
            header = obj;
          } else if (obj.type === 'msg') {
            const { type, ...msg } = obj; // 去掉 type 字段还原为 AIOpsMessage
            messages.push(msg);
          }
        } catch {
          // 跳过损坏行
        }
      }

      if (!header) return null;

      return {
        ...header,
        messageCount: messages.length,
        messages,
      };
    } catch {
      return null;
    }
  }

  /** 删除会话 */
  async deleteConversation(userId: string, fileName: string): Promise<boolean> {
    const filePath = path.join(this.getUserDir(userId), fileName);
    try {
      await fs.unlink(filePath);
      return true;
    } catch {
      return false;
    }
  }

  /** 辅助：只读文件第一行并解析为 JSON */
  private async readFirstLine(filePath: string): Promise<any> {
    return new Promise((resolve) => {
      const stream = fSync.createReadStream(filePath, { encoding: 'utf-8' });
      const rl = readline.createInterface({ input: stream });
      rl.on('line', (line) => {
        rl.close();
        stream.close();
        try {
          resolve(JSON.parse(line));
        } catch {
          resolve(null);
        }
      });
      rl.on('error', () => resolve(null));
      rl.on('close', () => resolve(null));
    });
  }

  /** 注册 IPC handlers */
  registerHandlers(): void {
    ipcMain.handle('chat-history:create', (_e, header) => this.createConversation(header));
    ipcMain.handle('chat-history:append', (_e, userId, convId, createdAt, message) =>
      this.appendMessage(userId, convId, createdAt, message));
    ipcMain.handle('chat-history:append-batch', (_e, userId, convId, createdAt, messages) =>
      this.appendMessages(userId, convId, createdAt, messages));
    ipcMain.handle('chat-history:update-header', (_e, userId, convId, createdAt, updates) =>
      this.updateHeader(userId, convId, createdAt, updates));
    ipcMain.handle('chat-history:list', (_e, userId) => this.listConversations(userId));
    ipcMain.handle('chat-history:load', (_e, userId, fileName) => this.loadConversation(userId, fileName));
    ipcMain.handle('chat-history:delete', (_e, userId, fileName) => this.deleteConversation(userId, fileName));
  }
}

export const chatHistoryService = new ChatHistoryService();
```

在 `main.ts` 中初始化：

### 4.3 Renderer 端 Service

```typescript
// src/services/chatHistoryService.ts

import { ConversationMeta, ConversationData, ConversationHeader } from '../types/chatHistory';

class ChatHistoryClientService {
  /** 创建新会话（写入 header） */
  async create(header: ConversationHeader): Promise<string> {
    return await window.electron.chatHistory.create(header);
  }

  /** 追加单条消息到会话文件末尾 */
  async appendMessage(userId: string, convId: string, createdAt: number, message: any): Promise<void> {
    await window.electron.chatHistory.append(userId, convId, createdAt, message);
  }

  /** 批量追加消息 */
  async appendMessages(userId: string, convId: string, createdAt: number, messages: any[]): Promise<void> {
    await window.electron.chatHistory.appendBatch(userId, convId, createdAt, messages);
  }

  /** 更新 header 元信息（如 updatedAt） */
  async updateHeader(userId: string, convId: string, createdAt: number, updates: Partial<ConversationHeader>): Promise<void> {
    await window.electron.chatHistory.updateHeader(userId, convId, createdAt, updates);
  }

  /** 获取会话列表（仅 header 信息） */
  async list(userId: string): Promise<ConversationMeta[]> {
    return await window.electron.chatHistory.list(userId);
  }

  /** 加载完整会话（header + 全部消息） */
  async load(userId: string, fileName: string): Promise<ConversationData | null> {
    return await window.electron.chatHistory.load(userId, fileName);
  }

  /** 删除会话文件 */
  async delete(userId: string, fileName: string): Promise<boolean> {
    return await window.electron.chatHistory.delete(userId, fileName);
  }
}

export const chatHistoryClientService = new ChatHistoryClientService();
```

## 5. 会话生命周期管理

### 5.1 useAIAgent hook 扩展

在 `useAIAgent.ts` 中新增会话管理能力。核心思路：**消息产生时实时追加写入，而非全量保存**。

```typescript
// 新增状态
const [convId, setConvId] = useState<string | null>(null);
const [convCreatedAt, setConvCreatedAt] = useState<number>(0);
const [convTitle, setConvTitle] = useState<string>('');
const [savedMsgCount, setSavedMsgCount] = useState(0); // 已持久化的消息数

// 记录已持久化消息数的 ref（供事件回调访问最新值）
const savedMsgCountRef = useRef(0);
savedMsgCountRef.current = savedMsgCount;

/** 发送消息时：自动创建会话 + 追加用户消息 */
async function sendMessage(input: string) {
  const now = Date.now();
  let currentConvId = convIdRef.current;

  // 首次发消息，创建新会话
  if (!currentConvId) {
    currentConvId = crypto.randomUUID();
    setConvId(currentConvId);
    setConvCreatedAt(now);
    setConvTitle(input.slice(0, 30));

    // IPC: 写入 header 行
    await chatHistoryClientService.create({
      convId: currentConvId,
      userId,
      hostId,
      hostName,
      title: input.slice(0, 30),
      mode,
      model: selectedModel,
      createdAt: now,
      updatedAt: now,
    });
  }

  // 构造用户消息
  const userMsg = { id: generateMessageId(), role: 'user', content: input, timestamp: now, ... };

  // IPC: 追加消息行
  await chatHistoryClientService.appendMessage(userId, currentConvId, convCreatedAt, serializeMsg(userMsg));
  setSavedMsgCount(prev => prev + 1);

  // ... 原有发送逻辑（setMessages、agent.ask 等）
}

/** task:complete 事件中：追加本轮产生的所有 assistant 消息 */
agent.on('task:complete', (summary, gemsRemaining) => {
  // ... 原有逻辑 ...

  // 找出尚未持久化的新消息（增量追加）
  const unsavedMessages = messagesRef.current.slice(savedMsgCountRef.current);
  if (unsavedMessages.length > 0 && convIdRef.current && userId) {
    const serialized = unsavedMessages.map(serializeMsg);
    chatHistoryClientService.appendMessages(userId, convIdRef.current, convCreatedAtRef.current, serialized);
    setSavedMsgCount(messagesRef.current.length);
  }

  // 更新 header 的 updatedAt
  if (convIdRef.current && userId) {
    chatHistoryClientService.updateHeader(userId, convIdRef.current, convCreatedAtRef.current, {
      updatedAt: Date.now(),
    });
  }
});

/** 新建会话（清空当前消息） */
function newConversation() {
  setMessages([]);
  setConvId(null);
  setConvCreatedAt(0);
  setConvTitle('');
  setSavedMsgCount(0);
}

/** 加载历史会话 */
function loadConversation(data: ConversationData) {
  setConvId(data.convId);
  setConvCreatedAt(data.createdAt);
  setConvTitle(data.title);
  setMessages(data.messages);
  setSavedMsgCount(data.messages.length); // 标记为全部已保存
}
```

### 5.2 消息序列化

追加前需剥离不可持久化的大数据（base64 附件内容）：

```typescript
// src/utils/chatHistoryUtils.ts

/** 序列化单条消息，剥离 base64 附件内容 */
function serializeMsg(msg: AIOpsMessage): any {
  return {
    id: msg.id,
    role: msg.role,
    content: msg.content,
    suggestion: msg.suggestion || undefined,
    taskState: msg.taskState || undefined,
    files: msg.files?.map(f => ({
      name: f.name,
      size: f.size,
      type: f.type,
      // 不保存 base64Content
    })),
    timestamp: msg.timestamp,
  };
}
```

### 5.3 持久化触发时机

| 触发点 | 操作 | 说明 |
|--------|------|------|
| `sendMessage()` | `create` + `append` | 首条消息时创建会话文件，同时追加用户消息行 |
| `task:complete` 事件 | `appendBatch` + `updateHeader` | AI 回答结束后，批量追加本轮 assistant 消息，更新 updatedAt |
| 用户点击"新建会话" | 无额外操作 | 消息已实时追加过，直接清空状态 |
| 加载历史会话 | 无额外操作 | 读取 .dat 文件，消息已全部持久化 |

## 7. 数据流总览

```
用户首次发送消息
    │
    ├─→ sendMessage()
    │   ├─→ 生成 convId（UUID）
    │   ├─→ IPC: chat-history:create → Main 写入 header 行
    │   └─→ IPC: chat-history:append → Main 追加 user 消息行
    │
AI 处理完成
    │
    ├─→ task:complete 事件
    │   ├─→ IPC: chat-history:append-batch → Main 追加 assistant 消息行
    │   └─→ IPC: chat-history:update-header → Main 更新 header 的 updatedAt
    │
用户继续追问
    │
    ├─→ sendMessage()
    │   └─→ IPC: chat-history:append → Main 追加 user 消息行（同一会话文件）
    │
用户点击"会话记录"
    │
    ├─→ IPC: chat-history:list
    │   → Main 读取每个文件第一行 header → 返回 ConversationMeta[]
    │   → 展示 ConversationList
    │
    ├─→ 用户选择某个会话
    │   → IPC: chat-history:load
    │   → Main 逐行解析 .dat 文件 → 返回 ConversationData
    │   → loadConversation() 恢复消息到 AI 面板
    │
用户点击"新建会话"
    │
    └─→ newConversation() → 清空消息、重置 convId
        （当前会话已实时追加保存，无需额外操作）
```

## 9. 注意事项

1. **文件大小控制**：单个 .dat 文件预计 10KB~500KB（取决于 agent 模式的执行输出）。JSONL 追加写入天然支持大文件增长，无需分片。
2. **并发写入**：JSONL 追加是原子性较好的操作（`fs.appendFile`），但同一会话的并发追加仍需在 Renderer 端做串行保证（Promise 队列或 mutex）。
3. **游客模式**：游客（无 userId）不支持会话保存，会话记录按钮隐藏。
4. **数据安全**：.dat 文件存储在用户 home 目录，权限跟随系统用户权限，不做额外加密。
5. **版本迁移**：通过 header 行的 `version` 字段支持未来格式变更，读取时检查版本号并做相应转换。
6. **附件处理**：base64 内容不持久化，只保存文件名、大小、类型等元信息，避免磁盘膨胀。
7. **损坏容错**：逐行解析，单行 JSON 损坏只影响该条消息，其余消息可正常恢复。
8. **列表性能**：读取会话列表只需读每个文件第一行（stream + readline），不加载全部消息内容，即使文件很大也能快速响应。
