# TermCat 插件开发流程

> 本文档介绍如何从零开始开发 TermCat 插件，涵盖内置插件和外部插件两种模式。

---

## 插件类型对比

| 特性 | 内置插件 | 外部插件 |
|------|---------|---------|
| 运行环境 | Renderer 进程 | Main 进程（Node.js） |
| UI 渲染 | React 组件 | 模板驱动（JSON → 预定义模板） |
| 样式系统 | 共享 Tailwind CSS + CSS 变量 | 模板自带主题色 |
| 权限控制 | 无需声明（应用自身的一部分） | 需在 package.json 中声明 |
| 安装方式 | 编译进应用 | 从 URL 下载 .tgz 安装 |
| 国际化 | 独立 locales + `createPluginI18n` | `contributes.i18n` 或自行管理 |
| 适用场景 | 核心功能模块（文件浏览、AI 运维等） | 第三方扩展 |

---

## 一、内置插件开发流程

### Step 1: 创建插件目录

在 `src/plugins/builtin/` 下创建新目录：

```
src/plugins/builtin/my-plugin/
├── index.ts           # 插件入口（导出 BuiltinPlugin）
├── i18n.ts            # 国际化 hook
├── locales/
│   ├── index.ts       # 翻译入口
│   ├── zh.ts          # 中文翻译
│   ├── en.ts          # 英文翻译
│   └── es.ts          # 西班牙语翻译
└── components/
    └── MyPanel.tsx     # React 组件
```

### Step 2: 定义翻译

```typescript
// locales/zh.ts
export const zh = {
  displayName: '我的插件',
  description: '插件功能描述',
  tabTitle: '我的面板',
  // ... 组件内文本
};

// locales/index.ts
import { zh } from './zh';
import { en } from './en';
import { es } from './es';
export const locales = { zh, en, es };

// i18n.ts
import { createPluginI18n } from '../utils/create-plugin-i18n';
import { locales } from './locales';
import { zh } from './locales/zh';

export const { useT, getLocale } = createPluginI18n(locales, zh);
```

**`createPluginI18n` 返回：**
- `useT()` — React Hook，在组件中获取当前语言翻译
- `getLocale(lang)` — 普通函数，在 `activate()` 等非组件上下文中使用

### Step 3: 实现插件入口

```typescript
// index.ts
import React from 'react';
import { FileText } from 'lucide-react';
import type { BuiltinPlugin, BottomPanelProps } from '../types';
import { MyPanel } from './components/MyPanel';
import { getLocale } from './i18n';

const MyPanelWrapper: React.FC<BottomPanelProps> = (props) => {
  return React.createElement(MyPanel, props);
};

export const myPlugin: BuiltinPlugin = {
  id: 'builtin-my-plugin',
  displayName: '我的插件',
  description: '插件功能描述',
  version: '1.0.0',
  getLocalizedName: (lang) => getLocale(lang).displayName,
  getLocalizedDescription: (lang) => getLocale(lang).description,

  activate(context) {
    // 注册底部面板
    context.registerBottomPanel({
      id: 'my-panel',
      title: 'My Panel',
      getLocalizedTitle: (lang) => getLocale(lang).tabTitle,
      icon: FileText,
      priority: 40,
      component: MyPanelWrapper,
    });

    // 监听连接变化
    context.onConnectionChange((info) => {
      if (info) {
        console.log('连接到:', info.hostname);
      }
    });
  },

  deactivate() {
    // 清理工作（subscriptions 已自动 dispose）
  },
};
```

### Step 4: 注册插件

在 `src/plugins/builtin/index.ts` 中注册：

```typescript
// 使用动态 import 实现懒加载
const loadMyPlugin = () =>
  import('./my-plugin').then(m => m.myPlugin);

export async function registerBuiltinPlugins() {
  const plugins = await Promise.all([
    // ... 已有插件
    loadMyPlugin(),
  ]);
  plugins.forEach(p => builtinPluginManager.register(p));
}
```

### Step 5: 实现 React 组件

```typescript
// components/MyPanel.tsx
import React from 'react';
import type { BottomPanelProps } from '../../types';
import { useT } from '../i18n';

export const MyPanel: React.FC<BottomPanelProps> = ({
  connectionId,
  theme,
  isVisible,
}) => {
  const t = useT();

  if (!isVisible) return null;

  return (
    <div className="h-full overflow-auto p-3">
      <h3 className="text-sm font-medium text-[var(--text-primary)]">
        {t.tabTitle}
      </h3>
      {/* 组件内容 */}
    </div>
  );
};
```

### Step 6: 事件通信（可选）

如果需要与宿主或其他插件通信：

```typescript
// 在 plugins/builtin/events.ts 中定义事件常量
export const MY_PLUGIN_EVENTS = {
  DATA_SELECTED: 'myPlugin:dataSelected',
} as const;

// 插件中发送事件
context.emitEvent(MY_PLUGIN_EVENTS.DATA_SELECTED, { id: 'xxx' });

// 宿主或其他插件监听
builtinPluginManager.on(MY_PLUGIN_EVENTS.DATA_SELECTED, (payload) => {
  // 处理事件
});
```

> **注意：** 宿主代码（App.tsx、TerminalView.tsx 等）禁止直接 import 插件模块，应从 `events.ts` 导入事件常量，避免破坏懒加载。

## 二、外部插件开发流程

### Step 1: 初始化插件项目

```bash
mkdir termcat-plugin-example
cd termcat-plugin-example
npm init -y
npm install typescript --save-dev
```

### Step 2: 配置 package.json

```json
{
  "name": "termcat-plugin-example",
  "version": "1.0.0",
  "description": "TermCat 示例插件",
  "termcat": {
    "displayName": "示例插件",
    "description": "这是一个 TermCat 示例插件",
    "entry": "dist/extension.js",
    "activationEvents": ["onStartup"],
    "permissions": [
      "terminal.read",
      "terminal.write",
      "ssh.exec",
      "monitor.read"
    ],
    "contributes": {
      "commands": [
        { "id": "example.hello", "title": "Say Hello" }
      ],
      "settings": {
        "refreshInterval": {
          "type": "number",
          "default": 5000,
          "description": "数据刷新间隔 (ms)"
        }
      }
    }
  }
}
```

### Step 3: 实现插件入口

```typescript
// src/extension.ts
import type { PluginContext } from '@termcat/plugin-types';

export function activate(context: PluginContext & { api: TermcatAPI }) {
  const { api } = context;

  // 注册命令
  api.commands.registerCommand('example.hello', () => {
    api.ui.showNotification('Hello from my plugin!', 'info');
  });

  // 注册状态栏项
  api.ui.registerStatusBarItem({
    id: 'example-status',
    text: 'Example',
    tooltip: 'Example plugin active',
    position: 'right',
    priority: 100,
  });

  // 注册模板面板
  api.ui.registerPanel({
    id: 'example-dashboard',
    title: 'Dashboard',
    icon: 'layout-dashboard',
    slot: 'sidebar-left',
    defaultSize: 280,
  });

  // 推送数据
  api.ui.setPanelData('example-dashboard', [
    {
      id: 'header',
      template: 'header',
      data: { title: 'Dashboard', icon: 'layout-dashboard' },
    },
    {
      id: 'metrics',
      template: 'metric-bars',
      data: {
        items: [
          { label: 'CPU', value: 45, unit: '%', color: 'primary' },
          { label: 'MEM', value: 62, unit: '%', color: 'warning' },
        ],
      },
    },
  ]);

  // 定时更新数据
  const timer = setInterval(async () => {
    const metrics = await api.monitor.getMetrics('current');
    api.ui.updateSection('example-dashboard', 'metrics', {
      items: [
        { label: 'CPU', value: metrics.cpu.usage, unit: '%', color: 'primary' },
        { label: 'MEM', value: metrics.memory.percentage, unit: '%', color: 'warning' },
      ],
    });
  }, 5000);

  context.subscriptions.push({ dispose: () => clearInterval(timer) });

  context.logger.info('Example plugin activated');
}

export function deactivate() {
  // 清理（subscriptions 会自动 dispose）
}
```

### Step 4: 编译打包

```bash
# tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "dist",
    "rootDir": "src",
    "strict": true
  }
}

# 编译
npx tsc

# 打包为 .tgz
npm pack
```

### Step 5: 安装测试

将 `.tgz` 文件放到可访问的 URL，在 TermCat 设置页面的插件管理中通过 URL 安装。

插件安装后的本地存储位置：
- macOS: `~/Library/Application Support/TermCat/plugins/<plugin-name>/`
- Windows: `%APPDATA%/TermCat/plugins/<plugin-name>/`
- Linux: `~/.config/TermCat/plugins/<plugin-name>/`

## 三、权限系统

外部插件需要在 `package.json` 中声明所需权限：

| 权限 | 说明 | 可访问的 API |
|------|------|-------------|
| `terminal.read` | 读取终端信息和数据 | `terminal.getTerminals()`, `terminal.onData()` 等 |
| `terminal.write` | 向终端写入数据 | `terminal.write()`, `terminal.executeCommand()` |
| `ssh.exec` | 执行 SSH 命令 | `ssh.exec()`, `ssh.registerConnectionInitHook()` |
| `files.read` | 读取远程文件 | `files.list()`, `files.read()` |
| `files.write` | 写入远程文件 | `files.write()` |
| `network` | 网络访问 | （预留） |
| `monitor.read` | 读取监控数据 | `monitor.getMetrics()`, `monitor.onMetricsUpdate()` |
| `ai.message` | AI 消息交互 | `ai.ask()`, `ai.onMessage()`, `ai.registerSlashCommand()` |
| `host.read` | 读取主机列表 | `host.getHosts()`, `host.getActiveHost()` |

未声明权限时调用对应 API 会抛出 `Error: Plugin "xxx" lacks permission: xxx`。

## 四、激活事件

插件可通过 `activationEvents` 声明何时激活：

| 事件 | 触发时机 |
|------|---------|
| `onStartup` | 应用启动时 |
| `onTerminalOpen` | 终端打开时 |
| `onTerminalData` | 终端有数据输出时 |
| `onSSHConnect` | SSH 连接建立时 |
| `onSSHDisconnect` | SSH 连接断开时 |
| `onFileOpen` | 文件被打开时 |
| `onFileSave` | 文件被保存时 |
| `onAIMessage` | AI 消息到达时 |
| `onCommand:<id>` | 执行指定命令时（支持 `*` 通配符） |
| `onView:<id>` | 打开指定视图时 |
| `onHostConnect:<id>` | 连接指定主机时 |

## 五、插件生命周期

```
发现 (discover)
  │
  ▼
已安装 (installed)
  │ 满足 activationEvent
  ▼
激活 (activated)
  │ activate(context) 被调用
  │ 插件注册 UI / 命令 / Hook
  │
  ▼
运行中...
  │ 用户禁用 / 应用退出
  ▼
停用 (deactivated)
  │ deactivate() 被调用
  │ subscriptions 自动 dispose
  │ 注册表清理
  ▼
卸载 (uninstalled) [仅外部插件]
  │ 本地目录删除
```

## 六、调试技巧

1. **日志输出**：外部插件通过 `context.logger` 输出日志，前缀为 `[Plugin:<pluginId>]`
2. **热重载**：修改外部插件代码后，重新安装即可，`require.cache` 会自动清除
3. **面板调试**：模板面板数据可通过 `setPanelData` 实时更新，无需重启
4. **内置插件调试**：直接在 Vite HMR 环境下开发，修改即生效
