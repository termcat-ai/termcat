# TermCat 插件工程结构

> 本文档描述 TermCat 插件系统的代码组织结构，包括核心框架、内置插件和外部插件的工程规范。

## 二、内置插件目录规范

每个内置插件遵循统一的目录结构：

```
src/plugins/builtin/<plugin-name>/
├── index.ts                # 插件入口，导出 BuiltinPlugin 对象
├── i18n.ts                 # 国际化 hook（useT + getLocale）
├── locales/
│   ├── index.ts            # 翻译入口（导出 locales 对象）
│   ├── zh.ts               # 中文翻译
│   ├── en.ts               # 英文翻译
│   └── es.ts               # 西班牙语翻译
├── components/             # React 组件
│   └── XXXPanel.tsx
├── hooks/                  # 自定义 Hook（可选）
│   └── useXxx.ts
├── services/               # 服务层（可选）
│   └── xxxService.ts
└── adapter/                # 数据适配器（可选）
    └── toXxx.ts
```

### 命名规范

| 项目 | 规范 | 示例 |
|------|------|------|
| 目录名 | kebab-case | `file-browser`, `ai-ops` |
| 插件 ID | `builtin-` 前缀 | `builtin-file-browser` |
| 导出变量 | camelCase + `Plugin` 后缀 | `fileBrowserPlugin` |
| 组件名 | PascalCase | `FileBrowserPanel` |
| 翻译文件 | 语言代码 | `zh.ts`, `en.ts`, `es.ts` |

### 现有内置插件

| 插件 ID | 功能 | UI 位置 | 优先级 |
|---------|------|---------|--------|
| `builtin-ai-ops` | AI 运维面板 | 右侧边栏 | — |
| `builtin-file-browser` | 文件浏览器 | 底部面板 | 10 |
| `builtin-transfer-manager` | 传输管理器 | 底部面板 | 20 |
| `builtin-command-library` | 快捷命令库 | 底部面板 | 30 |
| `builtin-monitoring-sidebar` | 系统监控 | 模板面板 (sidebar-left) | — |
| `builtin-demo-panel` | 模板 Demo | 模板面板 (sidebar-left) | 20 |

### 不可禁用插件

`builtin-ai-ops` 标记为不可禁用（`NON_DISABLEABLE`），其他内置插件可由用户在设置页中启用/禁用。

## 三、外部插件工程结构

### 最小结构

```
termcat-plugin-xxx/
├── package.json            # 包含 termcat 配置段
├── dist/
│   └── extension.js        # 编译后的入口文件（CommonJS）
└── README.md
```

### 完整结构

```
termcat-plugin-xxx/
├── package.json            # 包含 termcat 配置段
├── tsconfig.json
├── src/
│   ├── extension.ts        # 插件入口
│   ├── commands/           # 命令实现
│   ├── providers/          # Provider 实现
│   └── utils/
├── dist/
│   └── extension.js        # 编译产物
├── locales/                # 多语言（可选）
│   ├── zh.json
│   └── en.json
└── README.md
```

### package.json 配置段

```json
{
  "name": "termcat-plugin-xxx",
  "version": "1.0.0",
  "description": "插件描述",
  "termcat": {
    "displayName": "显示名称",
    "description": "插件描述",
    "entry": "dist/extension.js",
    "activationEvents": ["onStartup"],
    "permissions": ["terminal.read", "ssh.exec"],
    "contributes": {
      "commands": [],
      "menus": {},
      "sidebar": {},
      "settings": {},
      "i18n": {}
    }
  }
}
```

### 入口文件规范

入口文件必须导出 `activate` 函数，可选导出 `deactivate` 函数：

```typescript
// 必须
export function activate(context: PluginContext): void | Promise<void>;

// 可选
export function deactivate(): void | Promise<void>;
```

**PluginContext 字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `pluginId` | `string` | 插件 ID |
| `pluginPath` | `string` | 插件安装路径 |
| `subscriptions` | `Disposable[]` | 订阅列表（停用时自动清理） |
| `logger` | `PluginLogger` | 日志对象 |
| `storagePath` | `string` | 插件数据目录 |
| `api` | `PluginAPI` | 插件 API 代理（运行时注入） |

## 四、核心类职责

### PluginManager（Main 进程）

外部插件的完整生命周期管理：

- **发现**：扫描 `userData/plugins/` 目录，解析 `package.json` 中的 `termcat` 配置段
- **激活**：根据 `activationEvents` 懒加载插件模块，调用 `activate()`
- **API 代理**：为每个插件创建独立的权限受控 API 实例
- **IPC 桥接**：注册 IPC 处理器，转发 Renderer 请求
- **安装/卸载**：支持从 URL 下载 .tgz 安装，删除本地目录卸载

### BuiltinPluginManager（Renderer 进程）

内置插件的注册和生命周期：

- **注册**：收集所有内置插件定义
- **激活**：调用 `activate(context)`，创建 Context 提供注册 API
- **面板管理**：维护 sidebar/bottom/toolbar 面板注册表
- **连接推送**：将终端连接信息推送给所有插件
- **启用/禁用**：通过 localStorage 持久化用户的插件偏好

### PluginRegistry

全局注册表，收集所有插件注册的 Provider / Hook / UI 组件：

- 命令注册 (`commands`)
- Terminal 装饰器 / 补全 Provider
- SSH 连接初始化 Hook
- 文件预览 Provider / 右键菜单
- AI 消息预处理器 / 斜杠命令 / 预执行 Hook
- 监控指标收集器 / 告警规则
- 状态栏 / 工具栏 / 侧栏视图
- 主机装饰器
- 配置存储 / 插件存储

### PluginService（Renderer 进程）

Renderer 端的 IPC 通信服务，供 React 组件调用：

- 获取插件列表、启用/禁用插件
- 获取 UI 数据（状态栏、工具栏、右键菜单、斜杠命令）
- 执行插件命令
- 监听插件状态变化和通知

## 五、UI 贡献点系统架构

### 数据流

```
插件 → setPanelData(JSON) → panelDataStore → usePanelData(hook) → PanelRenderer → Templates
                                                                        ↓
                                                               React 组件渲染
```

### panelDataStore

面板数据的中央存储，基于发布-订阅模式：

```typescript
// 注册面板
panelDataStore.registerPanel(pluginId, options);

// 推送全量数据
panelDataStore.setPanelData(panelId, sections);

// 局部更新
panelDataStore.updateSection(panelId, sectionId, data);

// React Hook 订阅
const { registration, sections } = usePanelData(panelId);
```

### PanelRenderer

通用面板渲染器，根据 `SectionDescriptor` 分发到对应模板组件：

```
SectionDescriptor { template: 'header', data: {...} }
  → HeaderTemplate
SectionDescriptor { template: 'table', data: {...} }
  → TableTemplate
```

### 模板事件回调

模板组件通过 `onEvent` 回调向插件传递用户操作：

```typescript
// 模板中触发
onEvent?.('button:click', { buttonId: 'refresh' });

// 插件中监听（通过 panelDataStore 的事件机制）
```

## 六、IPC 通信通道

### 插件管理通道

| 通道 | 方向 | 说明 |
|------|------|------|
| `plugin:list` | R→M | 获取插件列表 |
| `plugin:enable` | R→M | 启用插件 |
| `plugin:disable` | R→M | 禁用插件 |
| `plugin:info` | R→M | 获取插件信息 |
| `plugin:install` | R→M | 安装插件 |
| `plugin:uninstall` | R→M | 卸载插件 |

### 插件 UI 数据通道

| 通道 | 方向 | 说明 |
|------|------|------|
| `plugin:statusbar:items` | R→M | 获取状态栏项 |
| `plugin:toolbar:buttons` | R→M | 获取工具栏按钮 |
| `plugin:sidebar:views` | R→M | 获取侧栏视图 |
| `plugin:file:contextmenus` | R→M | 获取文件右键菜单 |
| `plugin:ai:slashcommands` | R→M | 获取斜杠命令 |

### 插件事件通道

| 通道 | 方向 | 说明 |
|------|------|------|
| `plugin:state:changed` | M→R | 插件状态变化 |
| `plugin:statusbar:updated` | M→R | 状态栏更新 |
| `plugin:notification` | M→R | 插件通知 |
| `plugin:panel:register` | M→R | 面板注册（模板面板） |
| `plugin:panel:setData` | M→R | 面板数据推送 |
| `plugin:panel:updateSection` | M→R | 面板局部更新 |

## 七、外部插件安装存储

```
userData/
├── plugins/                    # 外部插件安装目录
│   ├── termcat-plugin-xxx/     # 每个插件一个目录
│   │   ├── package.json
│   │   └── dist/extension.js
│   └── ...
├── plugin-data/                # 插件数据目录
│   ├── termcat-plugin-xxx/     # 每个插件独立数据空间
│   └── ...
└── plugin-config.json          # 插件启用/禁用配置
```

`plugin-config.json` 格式：

```json
{
  "enabled": {
    "termcat-plugin-xxx": true,
    "termcat-plugin-yyy": false
  },
  "settings": {
    "termcat-plugin-xxx": {
      "refreshInterval": 3000
    }
  }
}
```
